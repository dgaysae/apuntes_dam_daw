En AWT se usaban **interfaces de pares**. Esto significaba que cada componente creado con AWT, creaba un par igual correspondiente al mundo real. Es decir: al crear, por ejemplo, un botón, se creaban en realidad dos: el botón virtual creado en Java y el que realmente se dibujaba en la pantalla (el real). El programador no necesita saber de la existencia de ese par, la comunicación del objeto creado con su par corría por cuenta de AWT.

Este modelo de componentes se elimina en Swing. En Swing se habla de **componentes de peso ligero** (lightweight). La clase `JComponent` que es la raíz de clases Swing, no utiliza un par, sino que cada componente es independiente del sistema de ventanas principal. Se dibujan a sí mismos y responden a los eventos de usuario sin ayuda de un par.

Los **contenedores**, como hemos dicho, son un tipo de componentes pensados para almacenar y manejar otros componentes. Por ejemplo, un cuadro de diálogo es un componente, pero a su vez contiene otros como cajas de texto, botones o barras de desplazamiento.

> [!abstract] Añadir componentes a un contenedor 
> Para hacer que un componente forme parte de un contenedor se utiliza el método **`add()`**.

> [!abstract] Eliminar componentes de un contenedor
> El método **`remove()`** es el encargado de eliminar un componente del contenedor. Tanto `add` como `remove` proceden de la clase **`java.awt.Container`** 

Además, Swing posee algunos **contenedores especiales** como: 

* **`JWindow`**. Representa un panel de ventana sin bordes ni elementos visibles. 

* **`JFrame`**. Objeto que representa una ventana típica con bordes, botones de cerrar, etc. 

* **`JPanel`**. Es la clase utilizada como contenedor genérico para agrupar componentes. 

* **`JDialog`**. Clase que genera un cuadro de diálogo.

Como todos ellos heredan de **[Container](https://docs.oracle.com/javase/8/docs/api/java/awt/Container.html)**, todos poseen dos métodos muy útiles para saber qué componentes hay en su interior:

* **`Component[] getComponents()`**: devuelve un array de los componentes que están dentro del contenedor.

* **`Component getComponentAt(int x, int y)`**: devuelve el componente que está ubicado en las coordenadas especificadas (calculadas dentro del contenedor).

Todos los contenedores derivan de una clase común, llamada `JComponent`, de la que heredan multitud de métodos útiles, además de los dos ya vistos de `Container`. Veamos, pues, en primer lugar, la clase `JComponent` y sus métodos.

Por ejemplo, un `JFrame` se puede construir de diversas maneras:

* `JFrame()`: Crea una ventana con la configuración normal.

* `JFrame(GraphicsConfiguration gc)`: Crea una nueva ventana usando la configuración gráfica indicada.

* `JFrame(String titulo)`: Crea una ventana con el título indicado.

* `JFrame(String titulo, GraphicsConfiguration gc)`: Crea una ventana con el título y la configuración gráfica indicada. 

Y, además, el mismo `JFrame` tiene un montón de métodos para manejarlo, como:

* **`void setDefaultCloseOperation(int modo)`**: Indica lo que ocurrirá al cerrar la ventana (haciendo clic en el aspa, pulsando ALT+F4, o por el método que sea). En "modo" se pueden suar cualquiera de estas constantes:

  * `JFrame.EXIT_ON_CLOSE` (terminar el programa: lo habitual)

  * `JFrame.DO_NOTHING_ON_CLOSE` (no hacer nada: el programa seguirá en ejecución, pero la ventana habrá quedad oculta). 

  * `JFrame.HIDE_ON_CLOSE` Oculta la ventana, pero no la cierra: puede volver a abrirse. 

  * `JFrame.DISPOSE_ON_CLOSE` Oculta la ventana y la destruye: no puede volver a abrirse. Apropiado, por ejemplo, para ventanas que no sean la ventana principal de la aplicación. 

* **`void setResizable(boolean b)`**: establece si la ventana podrá cambiar de tamaño o no. 

* **`void setTitle(String t)`**: establece el título de la ventana.

* **`String getTitle()`**: devuelve el título de la ventana.

* **`void pack()`**: ajusta la ventana al tamaño más ajustado a sus contenidos.

* **`void setState(int estado)`**: el estado puede ser `JFrame.NORMAL`, `JFrame.MAXIMIZED_HOR`, `JFrame.MAXIMIZED_VER`, `JFrame.MAXIMIZE_BOTH`, `JFrame.ICONIFIED`.

* **`void setLocation(int x, int y)`:** establece la ubicación de la ventana en la pantalla.

* **`setLocationRelativeTo(Component c)`**: sitúa la ventana ventana en relación a otra. Si le pasamos un `null` como argumento, la posicionará en el centro de la pantalla.

  > [!warning] ¡Cuidado! 
  > Debe llamarse siempre después de establecer el tamaño de la ventana (`setSize()` o `pack()`), de lo contrario aparecerá en la esquina superior izquierda y apenas será visible.

* **`void setSize(int x, int y`):** establece el tamaño de la ventana (`x` para el ancho e `y` para la altura).

* **`int getState()`**: devuelve el estado actual de la ventana.

El resto de contenedores tienen sus propios constructores y métodos. No vamos a presentar aquí una lista completa de estos métodos porque nos eternizaríamos. Puedes consultar la referencia completa en la documentación oficial de Java.

Veamos varios ejemplos prácticos de cómo crear una ventana en Java usando `JFrame`, ya que hereda de `Window` y, por tanto, nos ofrece toda su funcionalidad.

## JFrame - Nuestra primera pantalla en Swing

Probemos este código:

```java
import javax.swing.*;

public class Main {
    public static void main(String[] args) {
        JFrame frame = new JFrame("Mi primera ventana");
    }
}
```

Teóricamente, si instanciamos un objeto de tipo `JFrame`, al ejecutar este código deberíamos ver una ventana con el título "Mi primera ventana", ¿no?

Pero ¡maldita sea, no aparece nada!

Tranquilidad... Es normal, porque antes de crear una ventana hay que darle unas instrucciones mínimas:

1. Indicarle un **tamaño** (con el método `setSize(int x, int y)`).
2. Indicar a la ventana que se haga **visible**. Si, cuando creamos una ventana esta no se muestra en pantalla hasta que se lo indiquemos. Parece una tontería, pero tiene su lógica. Imagina que tu pantalla tiene muchos componentes, muy pesados, y que se muestra en pantalla y se va viendo cómo aparecen los componentes uno a uno. Admitamos que estaría feo.

Vamos a ello:

```java
import javax.swing.*;
import java.awt.*;

public class Main {
    public static void main(String[] args) {
        JFrame frame = new JFrame("Mi primera ventana");
        frame.setSize(400, 200);
        frame.setVisible(true);
    }
}
```

Y al ejecutarlo...

![ud06_swing_JFrame_02.png](/img/user/attach/Unidad%2006/ud06_swing_JFrame_02.png)

¡Eureka! Enhorabuena, ya tienes tu primera ventana. ¡Y muestra el título! 👏👏👏👏

Ahora intentar cerrarla... 😈

Efectivamente, si pulsas en el aspa (la "X") a la derecha de la barra de título de la ventana, no se cierra 😅

Esto, aunque fastidie un poco, también es normal. Piensa que una aplicación puede tener muchas ventanas. Algunas, al cerrarse, simplemente dejan de existir en el programa (que sigue funcionando). Otras provocarán el cierra completo de la aplicación cuando sean cerradas. Esto se hace con el método `setDefaultCloseOperation` que explicamos anteriormente.

En nuestro caso queremos que, cuando se cierre la ventana, la aplicación finalice su ejecución:

```java
import javax.swing.*;
import java.awt.*;

public class Main {
    public static void main(String[] args) {
        JFrame frame = new JFrame("Mi primera ventana");
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.setSize(400, 200);
        frame.setVisible(true);
    }
}
```

Si, ahora la cosa va sobre ruedas. Va tan bien que ¡qué demonios! vamos a poner un botón en ella:

```java
import javax.swing.*;
import java.awt.*;

public class Main {
    public static void main(String[] args) {
        JFrame frame = new JFrame("Mi primera ventana");
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.setSize(400, 200);
        
        // Introducimos un botón:
        JButton bt1 = new JButton("Aceptar");
        frame.add(bt1);
                
        /*
        Recuerda que lo último que haremos será mostrar la pantalla.
        Antes introduciremos todos sus componentes.
        */
        frame.setVisible(true);
    }
}
```

¡Y aquí lo tenemos! ¡Un enorrrrrrrrme botón que ocupa toda la ventana!

![ud06_swing_JFrame_03.png](/img/user/attach/Unidad%2006/ud06_swing_JFrame_03.png)

Efectivamente, falta otra cosa: indicarle el `LayoutManager`. Esta cosa tan rara indica a la ventana cómo organizar los elementos que introducimos en ella. Como no le hemos indicado ninguno, ha tomado el `LayoutManager` por defecto: `BorderLayout`.
