En esta sección haremos un repaso a los principales componentes de Swing (al margen de los contenedores, de los que ya hemos hablado). Veremos cómo manejar cuadros de diálogo, etiquetas, botones, cajas de texto y otros componentes habituales de los interfaces gráficos de usuario.

## 5.1. Cuadros de diálogo con `JOptionPane`

Una de las labores típicas en la creación de aplicaciones gráficas del tipo que sea, es la de comunicarse con el usuario a través de mensajes en forma de cuadro de diálogo. Algunos cuadros son extremadamente utilizados por su sencillez (textos de aviso, error, confirmación, entrada sencilla de datos, etc.). 

La clase **JOptionPane** deriva de JComponent y es la encargada de crear este tipo de cuadros. Aunque posee constructores, normalmente se utilizan mucho más una serie de métodos estáticos que permiten crear de forma más sencilla objetos JOptionPane. 

Los siguientes **métodos estáticos** son los más utilizados: 

* `static void showMessageDialog(Component padre, Object mensaje)`  
  Muestra un cuadro de diálogo en el contenedor padre indicado con un determinado mensaje 

* `static void showMessageDialog(Component padre,`  
      `Object mensaje, String título, int tipo)`  
  Muestra un cuadro de diálogo en el contenedor padre indicado con un determinado mensaje, título y tipo. 

* `static void showMessageDialog(Component padre,`  
      `Object mensaje, String título, int tipo, Icon i)`  
  Igual que el anterior pero se permite indicar un icono para acompañar el mensaje. 

En los casos en los que se puede indicar un **tipo**, éste puede ser:

* `JOptionPane.INFORMATION_MESSAGE`. Información.

* `JOptionPane.ERROR_MESSAGE`. Error.

* `JOptionPane.WARNING_MESSAGE`. Aviso. 

* `JOptionPane.QUESTION_MESSAGE`. Pregunta. 

* `JOptionPane.PLAIN_MESSAGE`. Sin icono.

Por ejemplo, el siguiente código mostrará un cuadro de diálogo como el de la figura:

 ![ud06_swing_JOptionPane_01.png](/img/user/attach/Unidad%2006/ud06_swing_JOptionPane_01.png)

```java
JOptionPane.showMessageDialog(this,
        "Soy un mensaje normal", 
        "Cuadro 1",
        JOptionPane.INFORMATION_MESSAGE);
```


Existen métodos estáticos semejantes a los anteriores para crear otros tipos de cuadros de diálogo: 

* **Cuadros de confirmación** con `showConfirmDialog()`. Por ejemplo:

  ```java
  int res = JOptionPane.showConfirmDialog(null, "¿Desea salir?", "Salir", 
                                         JOptionPane.YES_NO_OPTION,
                                         JOptionPane.QUESTION_MESSAGE); 
  if (res == JOptionPane.YES_OPTION) System.exit(0); 
  ```
 
  ![ud06_swing_JOptionPane_02.png](/img/user/attach/Unidad%2006/ud06_swing_JOptionPane_02.png)

* **Cuadros con entrada de datos** con `showInputDialog()`. Por ejemplo:

  ```java
  String nombre;
  String res = JOptionPane.showInputDialog("Escriba su nombre");
  if (res == null)
  	JOptionPane.showMessageDialog(
  			null,
  			"¡Debe escribir algo!");
  else
  	nombre = res;
  ``` 

  ![ud06_swing_JOptionPane_03.png](/img/user/attach/Unidad%2006/ud06_swing_JOptionPane_03.png)

* **Cuadros de diálogo modales** con `showInternalMessageDialog(), showInternalInputDialog() o showInternalConfirmDialog(`).

## 5.2. Etiquetas con `JLabel`

Las etiquetas son textos de una línea que sirven para dar información textual a las aplicaciones. También se pueden utilizar para mostrar imágenes estáticas. 

Aquí vemos un ejemplo de creación y colocación de una etiqueta:  

```java
ventana1 = new JFrame(); 
jp = new JPanel(false); 
l1 = new JLabel("Hola"); 
l2 = new JLabel("Adiós"); 


jp.add(l1); 
jp.add(l2); 
ventana1.setLocation(100, 100); 
ventana1.setSize(300, 300); 
ventana1.setContentPane(jp); 
ventana1.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); 
ventana1.setVisible(true);
```

Como se observa en el ejemplo, las etiquetas se pueden colocar con el método add de los paneles contenedores. 

Además, en una etiqueta se pueden colocar también **imágenes**. Las imágenes se basan en **un interfaz llamado Icon**. Para facilitar la tarea de usar iconos, se puede hacer uso de la **clase ImageIcon**. Se trata de una clase pensada para utilizar iconos a partir de ficheros de imagen.

He aquí un ejemplo:  

```java
JLabel grafica1 = new JLabel(new ImageIcon("D:/hola.gif"); 
JLabel grafica2 = new JLabel(
    "Texto de la etiqueta",
    new ImageIcon("D:/fondo.gif")
);
```

En la segunda etiqueta se mostrará una imagen y un texto.

## 5.5. Botones con JButton

Son elementos fundamentales en el trabajo de las aplicaciones. Cuando son pulsados, generan un evento ActionEvent que, capturado permite crear una acción para el botón.

La clase fundamental es **JButton,** que toma la mayor parte de su funcionalidad de AbstractButton, clase abstracta con la que se definen casi todos los botones. 

Cada ventana o cuadro de diálogo puede tener un **botón por defecto**. Este botón está asociado a la tecla Intro y aparece marcado de forma especial. Para hacer que un botón sea el botón por defecto usarmos el método setDefaultButton() de JRootPane: 

```java
...
JButton botonOk = new JButton("Aceptar");
JButton botonCancel = new JButton("Cancelar");
frame.add(botonOk);
frame.add(botonCancel);
...
```
   
![ud06_swing_JButton_01.png](/img/user/attach/Unidad%2006/ud06_swing_JButton_01.png)

**Manejando eventos de botón (y otros controles)** 

En cuanto a los **eventos**, tanto los botones y como los cuadros de texto (y otros controles) generan eventos del tipo **`ActionEvent`**. Para manipular estos eventos, se debe llamar, en cada control que queramos que lance eventos, al método `addActionListener`, indicando el objeto que manipulará estos eventos (este objeto debe pertenecer a alguna clase que implemente el interfaz `ActionListener`). Será el método `actionPerformed` el que se encargará de manipular el evento. 

Un problema típico consiste en que, a menudo, se necesita **averiguar qué botón o cuadro de texto lanzó el evento**. Una forma fácil de saberlo es mediante la cadena **`ActionCommand`**. Esta cadena es un texto que describe al objeto que lanzó el evento. Se usa de la siguiente forma: 

1. El objeto que lanza eventos de este tipo rellena su cadena `ActionCommand` usando su método `setActionCommand`

2. En el método `actionPerformed` podremos leer esa cadena usando el método `getActionCommand` del evento `ActionEvent` que se usa como argumento. 

Por ejemplo:

```java
...
JButton bt1 = new JButton("Aceptar");
bt1.setActionCommand("bt1"); // Reconoceremos a este botón por su ActionCommand
                             // Se puede usar cualquier cadena
...
```

En el método `actionPerformed`, el código sería: 

```java
public void actionPerformed(ActionEvent e){ 
    if (e.getActionCommand().equals("bt1")){ 
        System.out.println("Se pulsó Aceptar"); 
} 
``` 

Hablaremos más sobre los manejadores de eventos en un próximo apartado.

## 5.6. Recuadros de selección con JCheckBox

Se trata de controles que permiten su activación y desactivación a fin de elegir una serie de opciones independientes. En Swing es la clase `JCheckBox` la encargada de representarlas. Esta clase deriva de `JToggleButton`, que, a su vez, deriva de AbstractButton.

La siguiente imagen muestra una ventana con un `JLabel` ("¿Está de acuerdo con las condiciones del servicio?"), un `JButton` ("Continuar") y un `JCheckBox` ("Acepto")  

```java
import javax.swing.*;
import java.awt.*;
import java.awt.event.ItemEvent;
import java.awt.event.ItemListener;

public class EjemploCheckBox extends JFrame implements ItemListener {

    JCheckBox deportes, cultura;
    JLabel descripción;
    private String sdeportes = "", scultura = "";

    public EjemploCheckBox() {
        super("Mi Ventana");
        setSize(400,100);
    }

    public void VentanaCasillaAct(){
        getContentPane().setLayout(new FlowLayout());
        deportes = new JCheckBox("Deportes");
        cultura = new JCheckBox("Cultura");
        descripción = new JLabel("Tiene elegido: ");
        deportes.addItemListener(this);
        cultura.addItemListener(this);
        getContentPane().add(deportes);
        getContentPane().add(cultura);
        getContentPane().add(descripción);
        setVisible(true);
    }

    @Override
    public void itemStateChanged(ItemEvent e) {
        if (e.getItemSelectable() == deportes){
            if (e.getStateChange() == ItemEvent.SELECTED)
                sdeportes = " deportes";
            else sdeportes = "";
        }
        else { //sólo puede haberlo provocado el evento "Cultura"
            if (e.getStateChange() == ItemEvent.SELECTED)
                scultura = " cultura";
            else scultura = "";
        }
        descripción.setText("Tiene elegido: " + sdeportes + " " + scultura);
    }
    
    // Ejecutamos nuestro programa:
    public static void main(String[] args) {
        EjemploCheckBox ventana = new EjemploCheckBox();
        ventana.VentanaCasillaAct();
    }
}
```

## 5.7. Botones de opción con JRadioButton

Son casi iguales a los check boxes, pero se utilizan para elegir una opción de entre un grupo de opciones. Como las casillas de verificación, la clase JRadioButton encargada de crear botones de radio deriva de un tipo especial de botón, el JToggleButton, que a su vez deriva de JButton. Por lo tanto, los métodos comunes de JButton estarán disponibles.

A continuación reproducimos el mismo ejemplo que con los checkboxes, pero utilizando radio buttons agrupados en un `ButtonGroup`; así, será imposible seleccionar ambas casillas al mismo tiempo.  

```java
import javax.swing.*;
import java.awt.*;
import java.awt.event.ItemEvent;
import java.awt.event.ItemListener;

public class EjemploRadioButton extends JFrame implements ItemListener {
    JRadioButton deportes, cultura;
    ButtonGroup grupoBotones;
    JLabel descripción;
    private String sdeportes="",scultura="";

    public EjemploRadioButton() {
        super("Mi Ventana");
        setSize(400,100);
    }

    public void VentanaCasillaAct(){
        getContentPane().setLayout(new FlowLayout());

        // Creamos los radio buttons
        deportes = new JRadioButton("Deportes");
        cultura = new JRadioButton("Cultura");
        // Creamos el grupo y asignamos los dos radio buttons a él
        grupoBotones = new ButtonGroup();
        grupoBotones.add(deportes);
        grupoBotones.add(cultura);
        // Creamos la etiqueta
        descripción = new JLabel("Tiene elegido: ");

        // Añadimos los escuchadores de eventos a los radio buttons
        deportes.addItemListener(this);
        cultura.addItemListener(this);

        // Agregamos los radio buttons al panel y lo mostramos todo
        getContentPane().add(deportes);
        getContentPane().add(cultura);
        getContentPane().add(descripción);
        setVisible(true);
    }

    @Override
    public void itemStateChanged(ItemEvent e) {
        if (e.getItemSelectable()==deportes){
            if (e.getStateChange()==ItemEvent.SELECTED)
                sdeportes=" deportes";
            else sdeportes="";
        }
        else { //sólo puede haberlo provocado el evento "Cultura"
            if (e.getStateChange()==ItemEvent.SELECTED)
                scultura=" cultura";
            else scultura="";
        }
        descripción.setText("Tiene elegido: "+sdeportes+" "+scultura);
    }

    public static void main(String[] args) {
        EjemploRadioButton rb = new EjemploRadioButton();
        rb.VentanaCasillaAct();
    }
}
```

## 5.8. Barras de desplazamiento

### Viewports con JViewport

`JViewport` es la clase madre de todos los componentes que permiten desplazamientos (scrolls). Un viewport, por tanto, es una ventana dentro de la vista actual que muestra una sección de los datos y que permite desplazar la vista hacia el resto de datos. 

Un viewport se construye mediante un constructor por defecto. Una vez creado el objeto, necesitamos asignarle un componente contenedor (un `JPanel` normalmente) sobre el que actuará el viewport. Esa asignación se realiza mediante el método setView, al cual se le pasa como parámetro el componente que queremos visualizar mediante el viewport. 

### Barras de desplazamiento con JScrollPane

Es imposible hablar de barras de desplazamiento sin referirse a la clase **`JScrollPane`**. Se trata de una clase espectacular que permite **colocar barras de desplazamiento a cualquier componente**. Usa, al igual que Viewport, la interfaz Scrollable. 

### Barras de desplazamiento con JScrollBar

El último elemento para realizar los desplazamientos es la clase **`JScrollBar`**, que representa objetos de barra de desplazamiento. Normalmente es más que suficiente `JScrollPane` para controlar el scroll de un componente. No obstante, es posible utilizar las barras de desplazamiento para acciones interesantes o para modificar las propiedades de las barras de un `JScrollPane`. 

Las barras generan eventos del tipo **`AdjustmentEvent`** cuando se modifican los valores de las barras. El método que controla estos eventos es `AdjustmentEvent`. 

A continuación vemos un pequeño ejemplo de uso de barras de desplazamiento: 

```java
public class VentanaScrollBar extends JFrame implements AdjustmentListener {
    JLabel etiqueta;
    JScrollBar barra;

    // constructor
    public VentanaScrollBar() {
        setSize(200, 180);
        barra = new JScrollBar(JScrollBar.HORIZONTAL, 0, 30, 0, 300);
        etiqueta = new JLabel("Valor: 0");
        etiqueta.setHorizontalAlignment(JLabel.CENTER);
        getContentPane().add(barra, BorderLayout.NORTH);
        getContentPane().add(etiqueta, BorderLayout.SOUTH);
        barra.addAdjustmentListener(this);

        setVisible(true);
    }

    @Override
    public void adjustmentValueChanged(AdjustmentEvent e) {
        etiqueta.setText("Valor: " + e.getValue());
    }

    // Ejecutamos la ventana:
    public static void main(String[] args) {
        VentanaScrollBar ventanaScrollBar = new VentanaScrollBar();
    }
}
```

## 5.9. Deslizadores con JSlider

La clase `JSlider` representa un tipo de objeto similar a las barras de desplazamiento, pero pensado únicamente para elegir un valor numérico. Es una forma de evitar usar un cuadro de texto, donde el usuario puede teclear cualquier cosa, y cerrar el conjunto de valores de entrada a los ofrecidos por el slider. 

`JSlider` puede provocar **eventos `ChangeEvent`**, que deben ser capturados en clases que implementen la interfaz `ChangeListener`. Para que un deslizador lance eventos, debe usar el método `addChangeListener`. El método de captura de estos eventos es `stateChanged()`, que se producirá cuando se cambie de posición al deslizador. 

Aquí vemos un ejemplo: 

```java
public class VentanaSlider extends JFrame implements ChangeListener{ 

   JLabel etiqueta; 
   JSlider barra; 

   public VentanaSlider() { 
      barra = new JSlider(JSlider.HORIZONTAL, 0, 100, 0); 
      etiqueta = new JLabel("Valor: 0"); 
      etiqueta.setHorizontalAlignment(JLabel.CENTER); 
      barra.addChangeListener(this);
      
      //apariencia del slider 
      barra.putClientProperty("JSlider.isFilled", Boolean.TRUE); 
      barra.setPaintLabels(true); 
      barra.setPaintTicks(true); 
      barra.setMajorTickSpacing(25); 
      barra.setMinorTickSpacing(5); 
      getContentPane().add(barra,BorderLayout.NORTH); 
      getContentPane().add(etiqueta,BorderLayout.SOUTH); 
   } 


   public void stateChanged(ChangeEvent e) { 
      etiqueta.setText("Valor: " + ((JSlider) e.getSource()).getValue()); 
   }
} 
```

## 5.10. Listas desplegables con JList

Las listas son controles que permiten elegir entre un conjunto de alternativas. Permiten elegir una sola opción o varias (selección múltiple) si se hace clic manteniendo pulsada la tecla Control. 

La clase para crear todos estos tipos de listas es **`JList`**, que deriva directamente de `JComponent`. 

Por ejemplo, podemos crear una lista así: 

```java
JList animales = new JList(new String[]{"perro", "gato", "vaca", "oveja",
                "cerdo", "pollo","cabra","caballo","asno"});` 
```

A las listas se les pueden (y suelen) asignar **barras de desplazamiento**, como a cualquier otro componente. A nuestra lista de animales anterior, le crearíamos la barra de desplazamiento así: 

```java
JScrollPane sp = new JScrollPane(animales);
```

Las listas se controlan con **eventos ListSelectionEvent**, en el paquete `javax.swing.event`, que se lanzan en listas que hagan uso del método `addListSelectionListener`. Las clases que deseen controlar esos eventos deben implementar la interfaz `ListSelectionListener`, el cual obliga a definir el método `valueChanged()`, que será llamado cuando se modifique el valor de una lista. 

Por ejemplo:   

```java
public class VentanaLista extends JFrame implements ListSelectionListener {
    JList lista = new JList(new String[]{
            "perro", "gato", "vaca",
            "oveja", "cerdo", "pollo",
            "cabra", "caballo", "asno"
    });
    JLabel etiqueta = new JLabel("Seleccionado: ");
    JPanel p1 = new JPanel(), p2 = new JPanel();

    public VentanaLista() {
        setSize(250, 180);
        JScrollPane sp = new JScrollPane(lista);
        p1.setLayout(new FlowLayout());
        p2.setLayout(new FlowLayout());
        lista.setVisibleRowCount(5);
        lista.addListSelectionListener(this);
        p1.add(sp);
        p2.add(etiqueta);
        getContentPane().add(p1, BorderLayout.CENTER);
        getContentPane().add(p2, BorderLayout.SOUTH);
        setVisible(true);
    }

    public void valueChanged(ListSelectionEvent e) {
        String texto = "Seleccionado: ";
        int i;
        /*
        Se añade el texto del siguiente elemento
        seleccionado hasta el último
        */
        for (i = 0; i <= lista.getSelectedValues().length - 1; i++) {
            texto += (String) lista.getSelectedValues()[i] + " ";
        }
        etiqueta.setText(texto);
    }

    public static void main(String[] args) {
        VentanaLista ventanaLista = new VentanaLista();
    }
}
```

## 5.11. Cuadros combinados con JComboBox

Los cuadros combinados son listas especiales que combinan las capacidades de una lista y un cuadro de texto. En apariencia son cuadros de texto, pero un botón con una flecha permite desplegarlos para seleccionar una (y sólo una) opción. 

Al ser un control mixto, que puede editar texto o seleccionar un elemento de la lista, envía dos tipos de **eventos**: 

* **`ActionEvent`** (comentado anteriormente, sobre todo en el apartado de los botones) al pulsar Intro

* **`ItemEvent`** cuando se cambia un elemento de la lista (visto anteriormente en los controles de recuadro de verificación o checkbox) 

El método de evento `getStateChange()` permite saber si el cambio fue para seleccionar (`ItemEvent.SELECTED`) o para deseleccionar (`ItemEvent.DESELECTED`). No suele ser necesario usarlos ya que la clase `JComboBox` tiene elementos de sobra para trabajar. 

Mostramos a continuación el mismo código que en el ejemplo de la lista desplegable, pero en esta ocasión implementado con un cuadro combinado. 

```java
public class VentanaListaCombinada extends JFrame implements ItemListener {
    JComboBox lista = new JComboBox(new String[]{
            "perro", "gato", "vaca",
            "oveja", "cerdo", "pollo",
            "cabra", "caballo", "asno"
    });
    JLabel etiqueta = new JLabel("Seleccionado: ");
    JPanel p1 = new JPanel(), p2 = new JPanel();

    public VentanaListaCombinada() {
        setSize(250, 180);  // Dimensiones de la ventana
        p1.setLayout(new FlowLayout());
        p2.setLayout(new FlowLayout());
        lista.setMaximumRowCount(5);
        lista.addItemListener(this);
        p1.add(lista);
        p2.add(etiqueta);
        getContentPane().add(p1, BorderLayout.CENTER);
        getContentPane().add(p2, BorderLayout.SOUTH);

        setVisible(true);
    }

    @Override
    public void itemStateChanged(ItemEvent e) {
        etiqueta.setText("Seleccionado:" +
                (String) lista.getSelectedItem());
    }

    public static void main(String[] args) {
        VentanaListaCombinada frame = new VentanaListaCombinada();
    }
}
```

## 5.12. Cuadros de diálogo predefinidos

### Cuadros de diálogo genéricos con JDialog

La clase **`JDialog`** de Java permite crear cuadros de diálogo de propósito genérico. Esta es una de las clases más importantes de Swing. 

Puede crearse asignando como contenedor padre una ventana `JFrame`, por ejemplo. Es más versátil pero más difícil de manejar. Sus métodos vienen heredados de `JWindow`. Entre ellos, el método `show()` muestra el cuadro y el método `dispose()` lo cierra. 

Además de la clase genérica, existen clases de diálogo planteadas para un propósito específico. Vamos a ver dos de ellas muy utilizadas: el selector de color y el selector de archivos.

### Selector de color con JColorChooser

La clase `JColorChooser` permite obtener cuadros de selección de colores.

### Selector de archivos con JFileChooser

Otro diálogo muy habitual en las aplicaciones de escritorio es el selector de archivos, que nos permite navegar por el árbol de directorios local y seleccionar uno o varios archivos o directorios. La clase **`JFileChooser`** se utiliza para hacer todo esto. Su funcionamiento es muy parecido al selector de colores. Tiene un montón de métodos que le dan gran flexibilidad, y un par de métodos estáticos que permiten usarlo de forma sencilla.

Aquí vemos un ejemplo: 

```java
...
JFileChooser selector = new JFileChooser(); 
ImgFileFilter filtro = new ImgFileFilter(); 

selector.setFileFilter(filtro); 
int returnVal = selector.showOpenDialog(null); 

if(returnVal == JFileChooser.APPROVE_OPTION) { 
    System.out.println(
        "Se eligió el archivo "
        + selector.getSelectedFile().getName()
    ); 
}
...
```

### Filtros de archivo con FileFilter

Para que el ejemplo funcione, hay que definir la **clase `ImgFileFilter`**. Gracias a esa clase se consigue (como se verá en el siguiente ejemplo), que se muestren sólo archivos JPG y GIF. Eso es lo que se llama un **filtro**. 

Los **filtros** se crean usando clases derivadas de la **clase abstracta `FileFilter`**. Las clases de filtros deben incluir un método `accept()` que devuelva verdadero para todos los archivos que se deben mostrar en el cuadro y un método `getDescription()` que devuelva una cadena indicando el tipo de archivos que el filtro es capaz de mostrar. 

La definición sería algo como esto: 

```java
private class ImgFileFilter extends FileFilter{
    public boolean accept(File f) {
        if (f.getPath().endsWith(".jpg")
            || f.getPath().endsWith(".gif")
            || f.getPath().endsWith(".jpeg")
            || f.isDirectory())
            return true;
        else return false;
    }
    
    public String getDescription() {
        return "Archivos de imagen";
    }
}
```

El cuadro de diálogo resultante sería éste:

![ud06_swing_FileFilter_01.png](/img/user/attach/Unidad%2006/ud06_swing_FileFilter_01.png)

El archivo elegido con el cuadro de diálogo no se abre ni nada parecido. El cuadro de diálogo no tiene capacidad para hacer eso. Lo que sí hace es proporcionarnos el archivo seleccionado con `getSelectedFile()`, a partir de ahí será nuestro programa el que tiene que abrirlo y manipularlo. 

## 5.13. Menús con JMenuBar

Vamos a finalizar este recorrido por Swing mencionando brevemente los menús. Como son objetos más complejos, no vamos a profundizar en ellos tanto como en el resto de componentes, pero sí lo suficiente para que puedas construir los menús de tus aplicaciones gráficas sin problemas.

Swing distingue dos tipos de menú: el **menú de barra**, que es el típico menú de aplicación con las opciones del programa (Archivo, Edición, Ayuda, etc) y el **menú popup o menú de contexto**, que es el que se despliega haciendo clic con el botón derecho del ratón y cuya composición varía según en lugar donde hagamos clic (de ahí la denominación "de contexto")

Los menús de barra se implementan con la clase **`JMenuBar`**, y los menús popup con la clase **`JPopupMenu`**. Aquí nos referiremos a los primeros, aunque los segundos se manejan de forma similar.

Para crear un `JMenuBar` usaremos generalmente el constructor sin parámetros, así:

```java
barraMenu = new JMenuBar();
miFrame.setJMenuBar(barraMenu);
```

La segunda línea, con la invocación del método `setJMenuBar()` de `JFrame`, sirve para asociar el menú a un contenedor (típicamente, la ventana principal de nuestro programa)

Hay que tener en cuenta que un `JMenuBar` es un tipo especial de contenedor, así que dispone del método `add()` para añadir los diferentes menús, tales como "Archivo", "Edición", o lo que tu aplicación necesite. Empecemos, pues, por el principio: cómo añadir un menú a la barra de menús.

### JMenu

Cada uno de los menús individuales de la barra (ya sabes: el menú "Archivo", el menú "Edición", etc.) se crea instanciando un objeto de la clase `JMenu` y añadiéndolo al `JMenuBar`. Puedes ver como se hace en este ejemplo:

```java
barraMenu = new JMenuBar();   // Creamos la barra del menú
menuArchivo = new JMenu();    // Creamos un menú
menuEdicion = new JMenu();    // Creamos otro menú
barraMenu.add(menuArchivo);   // Añadimos el primer menú a la barra
barraMenu.add(menuEdicion);   // Añadimos el otro menú a la barra
miFrame.setJMenuBar(barraMenu);
```

El texto del menú se puede establecer con el constructor o empleando posteriormente el método `setText()`, así:

```java
menuArchivo = new JMenu("Archivo");  // Establece título en el constructor
menuEdicion = new JMenu();           // Constructor sin parámetros
menuEdicion.setText("Edición");      // Establece título con setText()
```

El `JMenu` tiene otros muchos métodos que pueden resultar interesantes. Por ejemplo, podemos asociar un atajo de teclado con `setMnemonic()`:

```java
menuArchivo = new JMenu("Archivo");
menuArchivo.setMnemonic(KeyEvent.VK_A);  // Atajo: tecla A
```

### Ítems de menú con JMenuItem

Los `JMenuItem` son las opciones del menú (tales como "Abrir", "Guardar", "Guardar como", etc. en el menú Archivo; o "Copiar", "Cortar", "Pegar", "Buscar" en el menú Edición)

Cuando creamos un `JMenu`, tendremos que crear también una colección de objetos `JMenuItem` decimos y asignarlos al `JMenu`:

```java
// Creamos el ítem de menú "Abrir" y lo añadimos al menú archivo
menuItemAbrir = new JMenuItem(); 
menuItemAbrir.setText("Abrir"); 
menuArchivo.add(menuItemAbrir); 

// Creamos el ítem de menú "Guardar" y lo añadimos al menú "Archivo"
// Esta vez, asignaremos el texto en el constructor del ítem
menuItemGuardar = new JMenuItem("Guardar");
menuArchivo.add(menuItemGuardar);
```

### Manejando los eventos de un menú

Un menú sin manejadores de eventos sería un menú inútil: los ítems se desplegarían pero no ocurriría nada cuando hiciésemos click sobre ellos.

Para que los ítems reaccionen a las pulsaciones con el ratón, tenemos que implementar el interfaz `ActionListener` y añadir un **escuchador de eventos** a cada ítem de menú con `addActionListener()`.  Además, como es lógico, tendremos que programar el código de captura del evento para que el ítem de menú haga lo que tiene que hacer:

```java
public class MenuDemo ... implements ActionListener {
    ...

    public MenuDemo() {
        // Creamos el JMenuBar, los JMenu y los JMenuItem
        // Para cada JMenuItem hay que hacer esto:
        menuItem.addActionListener(this);
        ...
    }
    
    public void actionPerformed(ActionEvent e) {
        // Aquí escribiremos el código necesario para reaccionar
        // al evento
    }
}
```

### JCheckBoxMenuItem y JRadioButtonMenuItem

Un tipo particular de ítem de menú es el JCheckBoxMenuItem. Son ítems de menú con una casilla de verificación asociada, de modo que pueden activarse y desactivarse, como los checkboxes convencionales.

El `JRadioButtonMenuItem`, no hará falta decirlo, es un radio button aplicado a un menú: nos permitirá activar una de entre un conjunto de opciones de menú.

Por lo demás, se crean exactamente igual que los ítems de menú normales:

```java
jCheckMenu1 = new JCheckBoxMenuItem("Check1");
jRadioButtonMenu1 = new JRadioButtonMenuItem("Radio1"); 
grupoRadios.add(jRadioButtonMenu1); 
menuOpciones.add(jRadioButtonMenu1);
```

> [!warning] Recuerda... 
> ...que los radio button y los checkboxes tienen sus propios eventos asociados, por lo que, para capturarlos, será necesario añadir los escuchadores apropiados (puede que no te sea suficiente con un `ActionListener`).
> Más información en http://docs.oracle.com/javase/8/docs/api

### Menús popup con JPopupMenu

Por ultimo, mencionaremos brevemente el `JPopupMenu`. A diferencia de los anteriores, este componente **no se asocia a una barra de menú, sino a un contenedor.** Puede ser un `JPanel` o cualquier otro contenedor, y por eso el menú popup puede ser diferente dependiendo de la zona de la aplicación donde se haga clic con el botón derecho del ratón.

Por lo demás, el uso del menú popup es igual al de un `JMenu` convencional:

```java
menuEmergente = new JPopupMenu();
itemEmergente1 = new JMenuItem("Opción emergente 1");
menuEmergente.add(itemEmergente1);
miPanel.add(menuEmergente); 
```

### Un ejemplo completo

Para unir toda la información vista someramente en esta sección, mostramos ahora un código de ejemplo de menú completo y funcional[^1]. Te puede servir de guía para construir tus propios interfaces basados en menú, haciendo los cambios necesarios.

El ejemplo crea dos menús, "File" y "Edit". El menú "File" tendrá una sola opción llamada "Open". El menú "Edit" contendrá "Copy", "Cut" y "Paste".

```java
import java.awt.*; 
import java.awt.event.*; 
import javax.swing.*; 
 
public class ejemploMenu implements ActionListener 
{ 
  private JFrame frame; 
  private JMenuBar menuBar; 
  private JMenu fileMenu; 
  private JMenu editMenu; 
  private JMenuItem openMenuItem; 
  private JMenuItem cutMenuItem; 
  private JMenuItem copyMenuItem; 
  private JMenuItem pasteMenuItem; 


  public void run() 
  { 
    frame = new JFrame("Java Menubar Example"); 
    menuBar = new JMenuBar(); 
    
    // Crea el menú File 
    fileMenu = new JMenu("File"); 
    openMenuItem = new JMenuItem("Open"); 
    openMenuItem.addActionListener(this); 
    fileMenu.add(openMenuItem); 
 
    // Crea el menú Edit 
    editMenu = new JMenu("Edit"); 
    cutMenuItem = new JMenuItem("Cut"); 
    copyMenuItem = new JMenuItem("Copy"); 
    pasteMenuItem = new JMenuItem("Paste"); 
    editMenu.add(cutMenuItem); 
    editMenu.add(copyMenuItem); 
    editMenu.add(pasteMenuItem); 
 
    // Añade los menús a la barra de menús 
    menuBar.add(fileMenu); 
    menuBar.add(editMenu); 
 
    // Asigna la barra de menús al JFrame 
    frame.setJMenuBar(menuBar); 
 
    frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); 
    frame.setPreferredSize(new Dimension(400, 300)); 
    frame.pack(); 
    frame.setLocationRelativeTo(null); 
    frame.setVisible(true); 
  } 
 
  /** 
   * Este método se encarga de manejar la acción sobre el menú File/Open, 
   * y muestra como se puede implementar un ActionListener. 
   */ 
  public void actionPerformed(ActionEvent ev) 
  { 
    SampleDialog dialog = new SampleDialog(); 
    dialog.setModal(true); 
    dialog.setVisible(true); 
  } 
 
  /** 
   * Este es el diálogo que se abre al seleccionar File/Open 
   */ 
  private class SampleDialog extends JDialog implements ActionListener 
  { 
    private JButton okButton = new JButton("OK"); 
 
    private SampleDialog() 
    { 
      super(frame, "Ejemplo de cuadro de diálogo", true); 
      JPanel panel = new JPanel(new FlowLayout()); 
      panel.add(okButton); 
      getContentPane().add(panel); 
      okButton.addActionListener(this); 
      setPreferredSize(new Dimension(300, 200)); 
      pack(); 
      setLocationRelativeTo(frame); 
    } 
 
    public void actionPerformed(ActionEvent ev) 
    { 
      setVisible(false); 
    } 
  } 
}
```

Podríamos usar la clase anterior desde este `main()`:

```java
public class ejemploMenuMain {
    public static void main(String[] args) {
        ejemploMenu menu = new ejemploMenu();
        menu.run();
    }
}
```
