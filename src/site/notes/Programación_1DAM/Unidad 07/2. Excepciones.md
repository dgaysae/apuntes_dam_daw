Hasta ahora, hemos manejado las excepciones cuando Java nos ha obligado a ello, pero quedan algunas cosas que decir sobre excepciones.

Para empezar, en Java existen dos tipos de excepciones:

* **Excepciones verificadas (checked exceptions):** Estas son las excepciones que el compilador de Java **obliga a manejar**. El programador **debe atrapar estas excepciones** utilizando un bloque `try-catch` (aunque hay una forma de evitarlo que veremos enseguida). Si no se maneja adecuadamente, el código no compilará. Ejemplos comunes de excepciones verificadas son `IOException`, `SQLException` o `ClassNotFoundException`.

* **Excepciones no verificadas (Unchecked exceptions):** Estas son las excepciones que no es obligatorio manejar. Son subclases de [**`RuntimeException`](https://docs.oracle.com/javase/8/docs/api/java/lang/RuntimeException.html)** y **no son chequeadas por el compilador**, lo que significa que el programador **no tiene que capturarlas obligatoriamente**. Ejemplos comunes de excepciones no verificadas son `NullPointerException`, `ArrayIndexOutOfBoundsException` o `ArithmeticException`.

## 2.1. Delegación de excepciones

Las excepciones verificadas (las que deben manejarse obligatoriamente) pueden **delegarse**. Es decir, un método que, en principio, está obligado a manejar una excepción puede decidir que sea otro método quien se encargue en realidad de hacerlo.

Por ejemplo, observa este código:

```java
public void metodo1() {
    try {
      metodo2();
    }
    catch (Exception e) {
        // Manejador de la excepción producida en metodo2
    }
}


public void metodo2() throws Exception {
    // Operaciones que pueden provocar una excepción
}
```

El `metodo2()` realiza operaciones que pueden provocar una **excepción verificada** y, en principio, debería tener un bloque `try-catch` para manejarla. Peeero observa la declaración del `metodo2()`. ¿Ves el `throws` de la cabecera o firma del método? Eso significa que este método no se rompe cuando salta una excepción. La **eleva**, es decir, está delegando el manejo de la excepción al método que lo invocó, es decir, en `metodo1()`.

Por eso en el `metodo1()` hay que incluir `metodo2()` en el bloque `try-catch`. Porque si saltara la excepción, `metodo2()` se la pasa `metodo1()` para que él se ocupe.

Al final, una excepción verificada debe manejarse en alguna parte, pero puede ser que nos convenga delegar ese manejo en otro método de orden superior.

## 2.2. Crear nuestras propias excepciones

Otra cosa interesante sobre las excepciones es que podemos **crear nuestra propia clase de excepciones** y lanzarlas cuando corresponda.

Observa este ejemplo:

```java
public class MiExcepcion extends Exception {  
      
    public MiExcepcion() {  
        super();   
    }

    public MiExcepcion(String mensaje) {  
        super(mensaje);  
    }  
    public String getMessage() {  
        return "Se ha producido una excepción de tipo MiExcepcion";  
    }  
    ...etc...   
}
```

La clase `MiExcepcion` hereda de `Exception` (como podría heredar de algún tipo concreto de excepción, como `IOException`, por ejemplo). De hecho, sus constructores llaman a los constructores de `Exception`. Podríamos añadir métodos adicionales para nuestra excepción si no necesitamos.

El ejemplo anterior podemos optimizarlo, conservando el mensaje que establece `getMessage()` pero sin tener que implementar dicho método:

```java
public class MiExcepcion extends Exception {  
      
    public MiExcepcion(String mensaje) {  
        super(mensaje);  
    }
    
    public MiExcepcion() {  
        this("Se ha producido una excepción de tipo MiExcepcion");   
    }  
    ...etc...   
}
```

En este caso, **`this`** representa al constructor `MiExcepcion(String mensaje)` pasándole el texto del mensaje, que lo envía al constructor de `Exception(super)`. Así también podemos establecer el mensaje de nuestra excepción.

Posteriormente, podemos crear métodos que, en caso de error de ejecución, lo manejen lanzando esta nueva excepción:

```java
public static void main(String[] args) {  
   try {  
      miMetodo();  
   } catch (MiExcepcion e) {  
      System.out.println(e.getMessage());  
   }  
}

*// Este método puede lanzar la excepción MiExcepcion*  
public static void miMetodo() throws MiExcepcion {  
   if (<condicion-de-error>)  
       throw new MiExcepcion("Algo ha ido mal!");  
}
```

En este ejemplo, el método `miMetodo()` lanza, bajo cierta condición de error, una excepción de tipo `MiExcepcion`, y el método `main()` captura y maneja dicha excepción, mostrando el mensaje de error que nosotros mismos hemos creado.
