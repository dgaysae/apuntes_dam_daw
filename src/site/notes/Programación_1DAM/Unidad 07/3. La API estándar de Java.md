El término API (_Application Program Interface_) se refiere a cualquier biblioteca de clases (o de funciones, en programación estructurada clásica) que ejerza de intermediario entre nuestros programas y algún otro sistema. Este otro sistema puede ser cualquier cosa: el sistema operativo, un gestor de bases de datos, otra aplicación o la máquina virtual de Java. Por eso un API puede ser casi cualquier cosa. 

Cuando hablamos del **API estándar de Java** nos referimos a la **biblioteca de clases que constituyen el JDK de Java** y que tenemos a nuestra disposición para usarlas en todas nuestras aplicaciones. 

la API de Java es extensísimo, y no es nuestra intención hacer aquí ni siquiera un resumen del mismo. Para eso ya están los [manuales de referencia](https://docs.oracle.com/javase/tutorial/getStarted/index.html) y el [sitio web de Oracle](https://docs.oracle.com/javase/8/docs/api/). Aquí contaremos cual es la forma habitual de proceder de un programador Java cuando necesita usar una o varias clases de la API de Java y no sabe o no recuerda cómo hacerlo. 


## 3.1. Clase `Date`

Supón que estás desarrollando una aplicación en la que necesitas trabajar con fechas y horas (algo bastante habitual, por otra parte). Parece lógico imaginar que la API de Java debe incluir alguna clase para trabajar con estos elementos tan frecuentes.

Una breve búsqueda por Internet nos revela que existen un buen número de clases en la API de Java para hacer esto. Lo siguiente debería ser ir a la web de Oracle y revisar la referecia de todas ellas. Encontraremos algo así:

* **Clase `Date`**: nos permite manejar la fecha en formato Unix, es decir, representada como el número de milisegundos transcurridos desde el 1 de enero de 1970, fecha que se toma como referencia (instante 0) en los relojes internos de todos los ordenadores del mundo.

* **Clase `Calendar`**: clase abstracta que define métodos para obtener la fecha y la hora como años, meses, días, horas, minutos y segundos. Como es abstracta, no puede instanciarse directamente.

* **Clase `GregorianCalendar`**: deriva de `Calendar`, y contiene la implementación de los métodos de `Calendar` para el calendario Gregoriano que utilizamos en Occidente. 

Por lo tanto, pronto se hace claro que `GregorianCalendar` es la clase que buscamos. Si miramos un poco la API de esta clase encontraremos algunas peculiaridades: 

* Los años, meses, etc, se representan con números enteros.
* Se accede a estos datos con constantes cuyos nombres son:
  - `DAY_OF_WEEK`
  - `DAY_OF_MONTH`
  - `MONTH`
  - `YEAR`
  - `HOUR`
  - `MINUTE`
  - `SECOND`
  - `MILISECOND` 
  - Etc. 

* Los días se la semana empiezan a contarse desde el domingo (domingo = 1) 

* Los meses del año empiezan a contarse desde 0 (enero = 0 y diciembre = 11) (sí, sí, has leído bien, enero es 0 y diciembre es 11. ¿Que por qué? Porque Java lo vale)

* Antes de usar `GregorianCalendar`, hay que instanciar `Date` para obtener la fecha en formato Unix. 

Por lo tanto, enseguida podemos hacer uso de `GregorianCalendar` para, por ejemplo, obtener la fecha y hora actual del sistema:

```java
import java.util.*; 
... 
GregorianCalendar calendario = new GregorianCalendar(); 
calendario.setTime(new Date()); 
int dia = calendario.DAY_OF_MONTH; 
int mes = calendario.MONTH; 
int ano = calendario.YEAR; 
int hora = calendario.HOUR; 
int minuto = calendario.MINUTE; 
int segundo = calendario.SECOND; 
System.out.println(
    "Fecha y hora actual: "
    + dia + "/" + mes + "/" + ano
    + " - " + hora + ":" + minuto + ":" + segundo);
... 
```

## 3.2. Una visión más actual para tratar con fechas

Java 8 introdujo una API completamente nueva y robusta para el manejo de fechas y horas, conocida como [**JSR 310**](https://jcp.org/aboutJava/communityprocess/pfd/jsr310/JSR-310-guide.html), en el paquete **`java.time`**. La intención era ofrecer más facilidades a la hora de manejar fechas, con clases más específicas y métodos más intuitivos que los que ofrecían las clases **`Date`** y **`Calendar`** explicadas en el apartado anterior.

Por ejemplo, las clases de java.time generan objetos **inmutables** que permiten a los hilos operar de forma segura (*thread-safe*) sin pisotearse unos a otros. También hay clases dedicadas exclusivamente a distintos tipos de representaciones temporales: sólo fechas, sólo horas, fechas y horas combinadas, zonas horarias...

Los datos son más coherentes que con **`Date`** y **`Calendar`** que, como decíamos antes, representaban los meses desde el 0 (enero) al 11 (diciembre), mientras que en java.time se hacen desde el 1 al 12.

Y si hablamos de cómo tratar fechas y hacer cálculos con ellas... Por todos estos motivos el paquete java.time es el **estándar recomendado** para el tratamiento de datos temporales.

Como hemos comentado antes, las clases de `java.time` ofrecen objetos **inmutables**. Esto quiere decir que cualquier modificación **no cambia el valor del objeto**, sino que devuelve el valor modificado en una nueva instancia. Algo así ocurre con la clase **`String`**, que ya conocemos de unidades anteriores.

Esto la convierte en una **API fluida** (*fluent API*). Es decir, está diseñada de tal forma que se pueden encadenar llamadas a métodos para hacer el código más legible. En otras palabras, permite leer el código casi como si fuese expresado en lenguaje natural.

> [!note] Nota 
> La generación de objetos en estas clases se basa en el patrón **`Builder`**.

### 3.2.1. Fechas

Para tratar una fecha sin más datos que día, mes y año tenemos la clase java.time.**`LocalDate`**.

Los objetos instanciados a partir de esta clase representan una fecha que se expresa normalmente en formato **año-mes-día**. Se pueden instanciar objetos de fecha (`LocalDate`) con builders de la propia clase:

```java
// Instanciamos un objeto LocalDate para la fecha 09/08/2025
LocalDate fechaConcreta = LocalDate.of(2025, 8, 9);
System.out.println("La fecha indicada es: " + fechaConcreta);


// Instanciamos un objeto fecha para hoy
LocalDate hoy = LocalDate.now();
System.out.println("Fecha de hoy: " + hoy);


// Instanciamos un objeto fecha para dentro de una semana
LocalDate enUnaSemana = LocalDate.now().plusDays(7);
System.out.println("Dentro de una semana: " + enUnaSemana);
// O...
enUnaSemana = LocalDate.now().plusWeeks(1)
System.out.println("Dentro de una semana: " + enUnaSemana);


// Instanciamos un objeto fecha para dentro de tres meses
LocalDate enTresMeses = LocalDate.now().plusMonths(3);
System.out.println("Dentro de tres meses: " + enTresMeses);
```

El que sean inmutables significa que no pueden cambiar de valor. Esto es, para que el objeto `fechaConcreta` cambie de valor, se le debe asignar otra instancia. Cualquiera de los métodos que se van a explicar a continuación no varía el valor de la fecha de un objeto. Simplemente devuelven otro objeto `LocalDate` con el nuevo valor.

Veamos algunos métodos útiles para el manejo de fechas:

```java
...
// Obtenemos la fecha de hoy
LocalDate hoy = LocalDate.now();

// Realizar operaciones con fechas
// Sumamos una semana:
LocalDate proximaSemana = hoy.plusWeeks(1);
System.out.println("La próxima semana: " + proximaSemana);

// Restamos un año:
LocalDate elAnioPasado = hoy.minusYears(1);
System.out.println("El año pasado: " + elAnioPasado);

// Formatear una fecha
LocalDate miFecha = LocalDate.of(2025, 10, 6);
// Instanciamos un formato de fecha mes/dia/año:
DateTimeFormatter formato = DateTimeFormatter.ofPattern("MM/dd/yyyy");

// Aplicamos el formato a la fecha:
String fechaFormateada = miFecha.format(formato);
System.out.println("Fecha formateada: " + fechaFormateada);

// Nº de días entre dos fechas
LocalDate fechaInicio = LocalDate.of(2025, 9, 12);
LocalDate fechaFin = LocalDate.of(2025, 11, 30);
long diasEntre = ChronoUnit.DAYS.between(fechaInicio, fechaFin);
System.out.println("Días entre las fechas: " + diasEntre);

// Determinar si un año es bisiesto
LocalDate fechlaCumple = LocalDate.of(2025, 11, 30);
boolean esBisiesto = fechlaCumple.isLeapYear();
System.out.println("¿Es bisiesto el año 2025? " + esBisiesto);
...
```

También se puede acceder a otros campos de fecha, como día del año, día de la semana, semana del año, etc.

```java
...
LocalDate fecha = LocalDate.of(2025, 11, 30);

System.out.println("Día del mes: " + fecha.getDayOfMonth());
System.out.println("Día de la semana: " + fecha.getDayOfWeek());
System.out.println("Día del año: " + fecha.getDayOfYear());
...
```

Se pueden crear fechas concretas de forma ***type-safe*** usando el enumerado `Month` para garantizar que se introduce un mes correcto:

```java
LocalDate hoy = LocalDate.of(2025, Month.NOVEMBER, 30);
```

Por otro lado, se pueden crear fechas a partir de cadenas de texto, si dicha cadena cumple con el formato estándar de la ISO (`yyyy-MM-dd`) a través del método `parse(String fechaISO)`:

```java
...
// Usando parse (si el formato es ISO 8601: yyyy-MM-dd)
LocalDate hoy = LocalDate.parse("2025-01-03");

System.out.println("Fecha de entrega: " + hoy);
// Salida: 2025-01-03
...
```

Hasta el momento podemos ver que las fechas resultantes se ven en el formato ISO 8601 pero ¿y si quisiéramos mostrarlas en formato español?

Para eso utilizamos la clase **DateTimeFormatter** del paquete java.time.format:

```java
import java.time.LocalDate;
import java.time.format.DateTimeFormatter;

public class EjemploFormatoFecha {
    public static void main(String[] args) {
        LocalDate fechaActual = LocalDate.now();
        
        // Definimos el patrón del formato:
        DateTimeFormatter formatoEspanol = DateTimeFormatter
                                .ofPattern("dd/MM/yyyy");
        DateTimeFormatter otroFormato = DateTimeFormatter
                                .ofPattern("dd-MM-yyyy");
        
        // Aplicamos el formato a la fecha en cuestión:
        String fechaFormateada = fechaActual.format(formatoEspanol);
        String fechaOtroFormato = fechaActual.format(otroFormato);
        
        System.out.println("Fecha en formato estándar (ISO): "
                + fechaActual);
        System.out.println("Fecha en formato español (dd/MM/yyyy): "
                + fechaFormateada);
        System.out.println("Fecha en otro formato similar (dd-MM-yyyy): "
                + fechaOtroFormato);
        
        // También funciona a la inversa: parsear una cadena no-ISO
        String fechaString = "30/09/2025";
        LocalDate fechaParseada = LocalDate.parse(fechaString, formatoEspanol);
        System.out.println("Fecha parseada: " + fechaParseada);
        // Salida: 2025-09-30 (LocalDate lo almacena como ISO)
        
        fechaParseada = LocalDate.parse(fechaString, otroFormato);
        System.out.println("Fecha parseada: " + fechaParseada);
        // Salida: 2025-09-30 (LocalDate lo almacena como ISO)
    }
}
```

Ahora que conocemos un poco mejor el uso de esta clase, veamos a qué se refiere el término ***fluent API*** con esta forma de declarar una fecha:

```java
...
LocalDate fechaFutura = LocalDate
       .now()
       .plusDays(2)
       .minusYears(10);
...
```

Como vemos, podemos encadenar varios métodos para obtener una nueva instancia para la fecha que tendría pasado mañana hace 10 años.

De la misma forma, los métodos no alteran el valor de la fecha sino que devuelven otra instancia:

```java
import java.time.LocalDate;
import java.time.Month;

public class EjemplosManipulacionFechas {
    public static void main(String[] args) {
        LocalDate hoy = LocalDate.now();

        // Sumar 10 días
        LocalDate futuro = hoy.plusDays(10);
        System.out.println("En 10 días: " + futuro);

        // Restar 3 meses
        LocalDate pasado = hoy.minusMonths(3);
        System.out.println("Hace 3 meses: " + pasado);

        // Obtener el primer día del mes, usando withDayOfMonth()
        LocalDate primerDia = hoy.withDayOfMonth(1);
        System.out.println("Primer día del mes: " + primerDia);
        
        // Comprobar si una fecha es anterior a otra
        boolean esAnterior = pasado.isBefore(hoy);
        System.out.println("¿Hace 3 meses es anterior a hoy? " + esAnterior);
    }
}
```

También se debe prestar atención a:

* [**`MonthDay`**](https://docs.oracle.com/javase/8/docs/api/java/time/MonthDay.html): según la documentación oficial, este objeto representa el par **mes-día** en el sistema de calendario ISO-8601. Si una fecha en formato ISO-8601 sería de la forma 2025-03-01, para un objeto `MonthDay` sería --03-01.
  
  Al igual que otras clases del paquete java.time, `MonthDay` es inmutable y representa la combinación de un mes y un día de dicho mes.
  ```java
  MonthDay nocheBuena = MonthDay.of(Month.DECEMBER, 24);
  System.out.println(nocheBuena);
  // --12-24
  ```

* [**`YearMonth`**](https://docs.oracle.com/javase/8/docs/api/java/time/YearMonth.html), de forma similar a `MonthDay`, representa el par año-mes. Para la fecha 2025-03-01, el objeto `YearMonth` guardaría 2025-03.
  ```java
  YearMonth diciembreEsteAnio = YearMonth
      .of(LocalDate.now().getYear(), Month.DECEMBER);
  System.out.println("Año y mes: " + diciembreEsteAnio);
  // Año y mes: 2025-12
  ```

### 3.2.2. Horas

Para tratar una hora (sólo la hora: hora, minutos, segundos...) se debe usar la clase java.time.**`LocalTime`**, que contiene **horas**, **minutos**, **segundos** y  **nanosegundos** (sin zonas horarias).

```java
// Obtenerhora actual usando el método estático now() en las clases locales:

import java.time.LocalTime;

public class EjemploHoraActual {
    public static void main(String[] args) {
        // Hora actual
        LocalTime ahora = LocalTime.now();
        LocalTime enDosHoras = ahora.plusHours(2);
        
        System.out.println("Hora actual: " + ahora);
        // Salida: 11:34:12.522234568
        
        System.out.println("Hora actual: " + ahora);
        System.out.println("dentro de 2 horas: " + enDosHoras);
        System.out.println("Hora: " + ahora.getHour());
        System.out.println("Minutos: " + ahora.getMinute());
        System.out.println("Segundos: " + ahora.getSecond());
        System.out.println("Nanosegundos: " + ahora.getNano());
        
        if (ahora.isBefore(enDosHoras)) {
            System.out.println(ahora + " es anterior a " + enDosHoras);
        }
        
        System.out.println(
            "Mismos minutos, segundos y nanosegundos pero con la hora 22: "
            + ahora.withHour(22));
    }
}
```


| Clase | Representación | Uso Típico |
| :---: | ----- | ----- |
| **`ZonedDateTime`** | Una **fecha y hora** con una **zona horaria** específica. | Reuniones internacionales, cálculo de la hora en diferentes países. |
| **`Instant`** | Un **punto instantáneo** en la línea de tiempo, típicamente en milisegundos desde el *Epoch* (1 de enero de 1970). | Medición de la duración, marcas de tiempo para persistencia. |

### 3.2.3. Fechas y horas combinadas

Con esta API es más fácil obtener y tratar la fecha/hora actual, crear una específica y manipularla. Se hace a través de la clase java.time.**`LocalDateTime`** Al igual que las anteriores, no tiene en cuenta la zona horaria.

```java
// Obtener fecha y hora actuales
import java.time.LocalDateTime;

public class EjemploFechaHoraActual {
    public static void main(String[] args) {
        // Fecha y hora actual combinadas
        LocalDateTime fechaYHora = LocalDateTime.now();
        System.out.println("Fecha y hora: " + fechaYHora);
        // Salida: 2025-12-03T13:11:34:12.83742246
    }
}
```

#### 3.2.3.1. Fechas y horas combinadas con zonas horarias

Si queremos aplicar una **zona horaria** a nuestros datos temporales, las clases [**`ZonedDateTime`**](https://docs.oracle.com/javase/8/docs/api/java/time/ZonedDateTime.html) y [**ZoneId**](https://docs.oracle.com/javase/8/docs/api/java/time/ZoneId.html) del paquete  java.time son la solución.

ZoneId nos ayuda a especificar la zona horaria. De esa forma podemos obtener la hora actual en una zona horaria particular, crear una fecha y hora estableciendo su zona horaria y convertir un dato temporal (fecha/hora) de una zona horaria en su semejante en otra zona horaria distinta.

Si, por ejemplo, queremos obtener la fecha y hora actuales en otra zona horaria:

```java
import java.time.ZonedDateTime;
import java.time.ZoneId;


// Obtiene la hora actual en la zona horaria de Nueva York
ZonedDateTime ahoraEnNuevaYork = ZonedDateTime
                .now(ZoneId.of("America/New_York"));
System.out.println("Hora en Nueva York: " + ahoraEnNuevaYork);
// 2025-12-05T06:02:04.238513294-05:00[America/New_York]
```

Para crear una fecha y hora determinada para zona horaria:

```java
import java.time.ZonedDateTime;  
import java.time.ZoneId;

ZonedDateTime fechaHoraEvento = ZonedDateTime  
                .of(2025, 12, 31, 23, 0, 0, 0, ZoneId.of("Europe/Madrid"));

System.out.println("Evento en Madrid: " + fechaHoraEvento);
```

Para convertir una fecha de una zona horaria a su equivalente en otro zona horaria diferente:

```java
import java.time.ZonedDateTime;
import java.time.ZoneId;

ZonedDateTime horaEnNuevaYork = ZonedDateTime
                .now(ZoneId.of("America/New_York"));

// Convierte la hora de Nueva York a la hora de Tokio
ZonedDateTime horaEnTokio = horaEnNuevaYork
                .withZoneSameInstant(ZoneId.of("Asia/Tokyo"));

System.out.println("Hora original en NY: " + horaEnNuevaYork);
// Hora original en NY: 2025-12-05T06:11:20.620608090-05:00[America/New_York]

System.out.println("Hora convertida en Tokio: " + horaEnTokio);
// Hora convertida en Tokio: 2025-12-05T20:11:20.620608090+09:00[Asia/Tokyo]
```

Veamos un ejemplo[^1] interesante para calcular la duración de un vuelo entre diferentes zonas horarias:

```java
ZoneId zonaHorariaDeSaoPaulo = ZoneId.of("America/Sao_Paulo");
ZoneId zonaHorariaDeNuevaYork = ZoneId.of("America/New_York");
 
LocalDateTime salidaDeSaoPauloSinZonaHoraria = LocalDateTime
                .of(2014, Month.APRIL, 4, 22, 30);
LocalDateTime llegadaANuevaYorkSinZonaHoraria = LocalDateTime
                .of(2020, Month.APRIL, 5, 7, 10);

ZonedDateTime salidaDeSaoPauloConZonaHoraria = ZonedDateTime
                .of(salidaDeSaoPauloSinZonaHoraria, zonaHorariaDeSaoPaulo);

System.out.println(salidaDeSaoPauloConZonaHoraria); //2020-11-04T22:30-03:00[America/Sao_Paulo]
 
ZonedDateTime llegadaANuevaYorkConZonaHoraria = ZonedDateTime
                .of(llegadaANuevaYorkSinZonaHoraria, zonaHorariaDeNuevaYork);

System.out.println(llegadaANuevaYorkConZonaHoraria); //2020-11-05T07:10-04:00[America/New_York]
	
Duration duracionDelVuelo = Duration
                .between(salidaDeSaoPauloConZonaHoraria,
                         llegadaANuevaYorkConZonaHoraria);

System.out.println(duracionDelVuelo); //PT9H40M
```

Si se hiciese el cálculo normal entre las horas de salida y llegada, el resultado sería de 8 horas y 40 minutos aproximadamente. Pero si se tiene en cuenta que durante el vuelo se viaja a otra zona horaria, habrá un cambio de hora correspondiente. Por eso se realiza dicho cálculo usando `ZonedDateTime` que dirá que realmente durará 9 horas y 40 minutos.

### 3.2.4. Instantes

La clase java.time.**`Instant`** sirve para almacenar un momento concreto en el tiempo con una precisión de nanosegundos. Se puede tomar como una marca de tiempo (*timestamp*) legible por máquina, **independiente de cualquier zona horaria o sistema de calendario humano**.

Internamente opera en UTC (Tiempo Universal Coordinado), basándose en el *Epoch* (1 de enero de 1970 a las 00:00:00 horas) de Unix . 

De esta forma, un objeto `Instant` nos permite tener un momento exacto y sin ambigüedades en la línea de tiempo. Esto viene muy bien si queremos representar el momento exacto en el que ocurre algo gracias a su precisión (en nanosegundos). Por ejemplo, registrar en un log lo ocurrido en un servidor. Esto **también ayuda a ver la duración de un suceso** comparando dos objetos `Instant`.

```java
Instant ahoraMismo = Instant.now();
System.out.println("Instante actual: " + ahoraMismo);
// Instante actual: 2025-12-05T12:06:02.021303209Z
```

Al igual que otras clases de java.time se pueden parsear instantes:

```java
String instanteString = "2025-12-05T12:06:02.021303209Z";
Instant instante = Instant.parse(instanteString);


System.out.println("Instante: " + instante);
// Instante: 2025-12-05T12:06:02.021303209Z
```

### 3.2.5. Periodos

La clase [**`Period`**](https://docs.oracle.com/javase/8/docs/api/java/time/Period.html) representa una cantidad de tiempo en términos de calendario: **años**, **meses** y **días**. Se utiliza principalmente para calcular la **diferencia entre dos fechas** (LocalDate) o para sumar/restar tiempo a una fecha específica.

Esta clase nos ofrece algunas formas de crear periodos:

* Con unidades específicas, usando métodos estáticos como ofYears(), ofMonths() u of():
  ```java
  import java.time.LocalDate;
  import java.time.Period;
  
  public class Periodos {
      public static void main(String[] args) {
          // Periodo de 2 años, 3 meses y 10 días
          Period p1 = Period.of(2, 3, 10);
          
          // Periodos de una sola unidad
          
          // Se convierte internamente a 35 días:
          Period cincoSemanas = Period.ofWeeks(5);
          Period dosAnios = Period.ofYears(2);
          ...
      }
  }
  ```

* Entre dos fechas, con el método between():
  ```java
  import java.time.LocalDate;
  import java.time.Period;
  
  public class Periodos {
      public static void main(String[] args) {
          ...
          // Calcular periodo entre dos fechas (muy útil para edades)
          LocalDate fechaNacimiento = LocalDate.of(2000, 5, 15);
          LocalDate hoy = LocalDate.now();
          Period edad = Period.between(fechaNacimiento, hoy);
          
          System.out.println("Tienes " + edad.getYears() + " años.");
      }
  }
  ```

El objeto `Period` se puede aplicar a objetos LocalDate usando los métodos plus() y minus():

```java
LocalDate hoy = LocalDate.now();
Period tiempoParaExamen = Period.ofMonths(1).plusDays(15);


LocalDate fechaExamen = hoy.plus(tiempoParaExamen);
System.out.println("El examen es el: " + fechaExamen);
```

> [!warning] Importante 
> Recordemos que `Period` es una clase **inmutable**. Los métodos **no cambian el objeto original**, sino que devuelven uno nuevo.

Algunos de los métodos más habituales para esta clase son:

* **getYears(), getMonths(), getDays()**: devuelven la cantidad de esa unidad específica.

* **withDays(int days)**: devuelve una copia con el número de días cambiado.

* **isNegative()**: indica si el periodo es negativo, es decir, si la fecha inicial es posterior a la final.

* **normalized()**: normaliza los meses. Por ejemplo, si el periodo contempla 13 meses, este método devuelve en 1 año y 1 mes.

Con este ejemplo podrás entenderlo mejor (y si juegas con él aprenderás más aún):  

```java
import java.time.LocalDate;  
import java.time.Period;  
import java.time.Month;

public class AnalisisAntiguedad {  
    public static void main(String[] args) {  
        LocalDate fechaContratacion = LocalDate.of(2018, Month.OCTOBER, 20);  
        LocalDate fechaActual = LocalDate.now();

        Period antiguedad = Period.between(fechaContratacion, fechaActual);

        System.out.println("El empleado lleva en la empresa:");  
        System.out.printf("%d años, %d meses y %d días.%n",   
            antiguedad.getYears(),   
            antiguedad.getMonths(),   
            antiguedad.getDays());  
              
        ***// Ejemplo de normalización***  
        Period pInestable = Period.of(0, 15, 40);   
        System.out.println("Periodo sin normalizar: " + pInestable);  
        System.out.println("Periodo normalizado: " + pInestable.normalized());  
    }  
}
```

### 3.2.6. Unidades de tiempo

#### 3.2.6.1. `ChronoUnit`

¡Y aquí vienen los enumerados! [**`ChronoUnit`**](https://docs.oracle.com/javase/8/docs/api/java/time/temporal/ChronoUnit.html) es un enum del paquete java.time cuyas constantes representan distintas unidades de tiempo: CENTURIES, DAYS, DECADES... incluso **FOREVER**.

Lo interesante de este enum (como de cualquier enum bien definido) es que proporciona métodos que nos ayudan a hacer cálculos y medir el tiempo en términos absolutos para cada una de estas unidades de tiempo.

Entre sus métodos más usuales está **between** que, dependiendo de la unidad de tiempo que indiquemos, nos dará la diferencia entre dos fechas medida en dicha unidad:

```java
import java.time.LocalDate;  
import java.time.Month;  
import java.time.temporal.ChronoUnit;

public class EjemploChronoUnit {  
    public static void main(String[] args) {  
        LocalDate inicioProyecto = LocalDate.of(2023, Month.JANUARY, 1);  
        LocalDate finProyecto = LocalDate.now();

        // Calcular la diferencia en distintas unidades  
        long dias = ChronoUnit.DAYS.between(inicioProyecto, finProyecto);  
        long semanas = ChronoUnit.WEEKS.between(inicioProyecto, finProyecto);  
        long meses = ChronoUnit.MONTHS.between(inicioProyecto, finProyecto);

        System.out.println("Tiempo que lleva activo el proyecto.");  
        System.out.println("Expresado en...");  
        System.out.println("...días? " + dias);  
        System.out.println("...semanas? " + semanas);  
        System.out.println("...meses? " + meses);  
    }  
}
```

Además de medir, sirve para **sumar o restar tiempo** a una fecha de forma muy legible usando el método **`addTo`** o simplemente pasándolo como argumento a `plus()`:

```java
LocalDate hoy = LocalDate.now();

// Sumar 3 décadas
LocalDate futuro = hoy.plus(3, ChronoUnit.DECADES);

// Restar 15 días
LocalDate pasado = hoy.minus(15, ChronoUnit.DAYS);
```

En el siguiente ejemplo[^2] podemos ver la versatilidad de todos estos elementos. Se trata de un ejercicio en el que el programa pide al usuario su fecha de nacimiento y calcula cuántos días lleva vivo y cuántas semanas faltan para su próximo cumpleaños:

```java
import java.time.LocalDate;
import java.time.temporal.ChronoUnit;
import java.util.Scanner;


public class CalculadoraVida {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        
        System.out.println("Introduce tu año de nacimiento (AAAA): ");
        int anio = sc.nextInt();
        System.out.println("Introduce tu mes de nacimiento (1-12): ");
        int mes = sc.nextInt();
        System.out.println("Introduce tu día de nacimiento: ");
        int dia = sc.nextInt();
        
        LocalDate fechaNac = LocalDate.of(anio, mes, dia);
        LocalDate hoy = LocalDate.now();
        
        // 1. Días totales de vida
        long diasVivo = ChronoUnit.DAYS.between(fechaNac, hoy);

        // 2. Semanas para el próximo cumple
        LocalDate proximoCumple = fechaNac.withYear(hoy.getYear());
        
        // Si ya pasó el cumple este año, calculamos para el año que viene
        if (proximoCumple.isBefore(hoy) || proximoCumple.isEqual(hoy)) {
            proximoCumple = proximoCumple.plusYears(1);
        }

        long semanasFaltan = ChronoUnit.WEEKS.between(hoy, proximoCumple);

        System.out.println("--- RESULTADOS ---");
        System.out.println("Llevas vivo exactamente "
                + diasVivo + " días.");
        System.out.println("Faltan "
                + semanasFaltan
                + " semanas para tu próximo cumpleaños.");
    }
}
```

#### 3.2.6.2. `Month`

¡Otro enumerado! [**`Month`**](https://docs.oracle.com/javase/8/docs/api/java/time/Month.html) representa los 12 meses del año (desde JANUARY hasta DECEMBER). En lugar de usar números, que pueden ser confusos (¿enero es 0 o 1?), usamos **constantes con nombres significativos**.

```java
import java.time.LocalDate;
import java.time.Month;
import java.time.format.TextStyle;
import java.util.Locale;


public class EjemploMonth {
    public static void main(String[] args) {
        // 1. Obtener Month desde un número (1-12)
        Month mesAgosto = Month.of(8);
        
        // 2. Obtener Month desde una fecha
        LocalDate hoy = LocalDate.now();
        Month mesActual = hoy.getMonth();
        
        // 3. Comparaciones y cálculos
        System.out.println("¿Es antes de agosto? "
        + mesActual.before(mesAgosto)); // Error común: Month no tiene before.
        // Corrección: Usamos el orden de la enumeración
        if (mesActual.getValue() < mesAgosto.getValue()) {
            System.out.println("Estamos antes de agosto.");
        }
        
        // 4. Sumar/Restar meses a la enumeración
        Month dentroDeTresMeses = mesActual.plus(3);
        System.out.println("En 3 meses será: " + dentroDeTresMeses);
    }
}
```

Una de las funciones más útiles es mostrar el nombre del mes en lenguaje natural.

```java
Month mes = Month.FEBRUARY;

// Nombre completo en español: "febrero"
System.out.println(mes.getDisplayName(TextStyle.FULL, new Locale("es", "ES")));

// Nombre corto: "feb."
System.out.println(mes.getDisplayName(TextStyle.SHORT, new Locale("es", "ES")));
```

Veamos lo que se puede hacer con este enum con el siguiente ejercicio. Un programador tiene una guardia cada 3 meses. Si su primera guardia es el 15 de Enero de 2024, vamos a hacer un programa que calcule la fecha de su cuarta guardia y **muestre el nombre del mes** de esa cuarta guardia en español y **calcule cuántos días** faltan desde hoy hasta esa fecha:

```java
import java.time.LocalDate;
import java.time.Month;
import java.time.temporal.ChronoUnit;
import java.time.format.TextStyle;
import java.util.Locale;

public class GestionGuardias {
    public static void main(String[] args) {
        // Definimos la primera guardia usando el Enum Month
        LocalDate primeraGuardia = LocalDate.of(2024, Month.JANUARY, 15);
        
        // Calculamos la cuarta guardia (sumando 3 periodos de 3 meses = 9 meses)
        LocalDate cuartaGuardia = primeraGuardia.plusMonths(9);
        
        // 1. Obtener el mes de la cuarta guardia
        Month mesFinal = cuartaGuardia.getMonth();
        String nombreMes = mesFinal.getDisplayName(
                                TextStyle.FULL,
                                new Locale("es", "ES"));
        
        // 2. Calcular días restantes desde hoy (asumiendo hoy como referencia)
        long diasRestantes = ChronoUnit.DAYS.between(LocalDate.now(), cuartaGuardia);
        
        System.out.println("La cuarta guardia será el: " + cuartaGuardia);
        System.out.println("Mes: " + nombreMes);
        System.out.println("Días que faltan: " + diasRestantes);
    }
}
```

#### 3.2.6.3. `DayOfWeek`

El enum [**`DayOfWeek`**](https://docs.oracle.com/javase/8/docs/api/java/time/DayOfWeek.html) elimina cualquier confusión que pueda generar el uso de números para los días de la semana. Sigue el estándar [**ISO-8601**](https://www.iso.org/iso-8601-date-and-time-format.html), donde el **lunes es el día 1** y el **domingo el 7**.

Podemos obtener el día de la semana desde una fecha o mediante su valor numérico.

```java
import java.time.DayOfWeek;
import java.time.LocalDate;

public class Main {
    public static void main(String[] args) {
        // 1. Obtener el día de hoy
        LocalDate hoy = LocalDate.now();
        DayOfWeek diaDeHoy = hoy.getDayOfWeek();

        // Imprime el nombre en inglés (MONDAY, etc.)
        System.out.println("Hoy es: " + diaDeHoy);
        System.out.println("Valor numérico de hoy: " + diaDeHoy.getValue());

        // 2. Obtener por número (1 = Lunes, 7 = Domingo)
        DayOfWeek viernes = DayOfWeek.of(5);
    }
}
```

Al igual que [`Month`](https://docs.oracle.com/javase/8/docs/api/java/time/Month.html), permite calcular días hacia adelante o hacia atrás de forma circular. Esto es, si a un Domingo le sumamos 1, obtenemos Lunes.

```java
DayOfWeek martes = DayOfWeek.TUESDAY;
DayOfWeek cuatroDiasDespues = martes.plus(4); // SATURDAY
DayOfWeek tresDiasAntes = martes.minus(3);    // SATURDAY (da la vuelta al calendario)
```

Vamos a ver su utilidad con algunos ejemplos prácticos. Por ejemplo, comprobemos si es **fin de semana**.

```java
import java.time.DayOfWeek;
import java.time.LocalDate;

public class ValidadorLaboral {
    public static void main(String[] args) {
        LocalDate fecha = LocalDate.of(2026, 3, 28); // sábado
        DayOfWeek dia = fecha.getDayOfWeek();

        if (dia == DayOfWeek.SATURDAY
            || dia == DayOfWeek.SUNDAY) {
            System.out.println(fecha + " cae en fin de semana (" + dia + ").");
        } else {
            System.out.println("Es día laborable.");
        }
    }
}
```


### 3.2.7. Antipatrones y excepciones

Cuando trabajamos con fechas, es fácil que salten excepciones en determinados escenarios.

#### 3.2.6.1. Unidad no soportada

Es la excepción más común y se manifiesta cuando intentamos pedirle a un objeto algo que no tiene. Por ejemplo, si le pedimos las horas o minutos a un objeto `LocalDate`.

La excepción es la `UnsupportedTemporalTypeException`:

```java
LocalDate hoy = LocalDate.now();
long horas = hoy.get(ChronoUnit.HOURS);   // ¡ERROR! LocalDate no tiene horas.
```

En estos casos podemos capturar o tratar la excepción o, si es necesario, usar otro tipo de objeto como `LocalDateTime` (que si tiene esas unidades).

#### 3.2.6.2. Fechas no válidas

Podríamos intentar construir fechas con datos que vienen del exterior (introducidas por teclado o por un formulario) y que no son válidos para una fecha correcta. En tal caso salta la `DateTimeException`:

```java
// ¡ERROR! El 31 de noviembre no existe.
LocalDate fechaMala = LocalDate.of(2023, 11, 31); 
```

En tales casos lo más sensato es capturarla con un `try-catch`.

#### 3.2.6.3. Cálculos arriesgados e innecesarios

Hacer sumas u otras operaciones con meses no es una ciencia exacta en días ya que unos tienen 30, otros 31 y febrero es un graciosillo cada cuatro años.

Por ejemplo, si a un **31 de enero** le sumamos un mes con `plusMonths(1)`, el resultado será el **28 o 29 de febrero** (según sea o no bisiesto). Si luego le restamos un mes a ese resultado, ¡volvemos al **28 o 29 de enero**, no al 31!

#### 3.2.6.4. Formatos de fechas incorrectos

Cuando convertimos un `String` a fecha usando `.parse()` estamos haciendo una apuesta casi segura a que salte la `DateTimeParseException`. Por ejemplo, metemos por teclado la fecha "12/10/2023" pero el programa espera el formato ISO "2023-10-12" que admite un objeto `LocalDate`.

Como antes, lo mejor es capturar la excepción y curarnos en salud.
