## 5.1. Expresiones lambda

Las **expresiones lambda** (a veces llamadas erróneamente *funciones anónimas*), son una característica introducida en Java 8 que permite escribir funciones anónimas de forma concisa. Son una herramienta muy poderosa para simplificar el código y mejorar la legibilidad en ciertas circunstancias.

Las expresiones lambda proporcionan **características de programación funcional** a Java. La programación funcional es un paradigma de programación que se basa en la ejecución de funciones ya existentes para resolver problemas en lugar de en la elaboración de algoritmos y en la interacción de objetos para resolver esos mismos problemas.

Es decir, la programación funcional persigue que el programador indique *qué quiere* calcular en lugar de escribir *cómo* calcularlo.

Este paradigma, que en realidad es muy antiguo (ya existía en la década de 1960 con el lenguaje Lisp), está de moda últimamente debido a la pujanza de lenguajes como Kotlin, JavaScript o [Python](https://www.python.org/), lenguajes multiparadigma que incorporan muchas características de programación funcional. Por eso, lenguajes más clásicos como [PHP](https://www.php.net/) o Java se han subido al carro y han añadido también sus propias características de programación funcional: las expresiones lambda.

No es nuestra intención impartir ahora un curso de programación funcional, pero vamos a mostrarte por encima qué es eso de las expresiones lambda. Fíjate en esta línea de código:

```
(x, y) -> x + y;
```

No parece Java, ¿verdad? Nadie lo diría, pero en esa línea se **está declarando una función**.

Esta expresión lambda define una función sin nombre que toma dos números como entrada (x e y) y devuelve la suma de ambos. El código equivalente en programación imperativa clásica sería este:

```java
public int sumar(int x, int y) { 
    return x + y;
}
```

Aquí tienes otro ejemplo de expresión lambda:

```java
// Ejemplo de expresión lambda: devuelve true si x es par
x -> x % 2 == 0;

// El mismo ejemplo, escrito de forma tradicional
public boolean esPar(int x) {
    return (x % 2 == 0);
}
```

Estas funciones, como cualquier otra función, pueden capturarse en una variable y usarse para lo que necesitemos:

```java
int x = 5;
boolean b1 = (x -> x % 2 == 0);
boolean b2 = esPar(x);
System.out.println(b1);           // Imprime “false”
System.out.println(b2);           // Imprime “false”

public boolean esPar(int x) {
    return (x % 2 == 0);
}
```

## 5.2. Interfaces funcionales, clases anónimas

Te voy a contar un secreto: **en realidad, Java no hace programación funcional, solo la simula**.

Es decir, cuando usamos una expresión lambda, *parece* que estamos escribiendo programación funcional, pero, por debajo, Java está instanciando un objeto de una clase, que es lo único que sabe hacer.

El secreto de Java para hacernos creer lo que no es es se llama *interface funcional*. Un **interface funcional** en Java es un tipo especial de interface caracterizado porque solo tiene un método abstracto. Por ejemplo:

`Operacion.java`
```java
@FunctionalInterface
interface Operacion {
    double calcular(double a, double b);
}
```

(La directiva @FuncionalInterface, por cierto, no es obligatoria, pero ayuda a que el compilador nos diga si estamos definiendo un interface funcional correcto, esto es, con un solo método abstracto).

Este interface funcional que hemos declarado (Operacion) puede implementarse ahora desde una clase tradicional de Java, por ejemplo la siguiente clase Suma:

```java
class Suma implements Operacion {
    @Override
    public double calcular(double a, double b) {
        return a + b;
    }
}
```

Una vez implementado el interface en la clase Suma, podemos proceder como con cualquier otra clase:

```java
Operacion miSuma = new Suma(); // Usamos una variable de tipo Operacion
int suma = miSuma.calcular(5, 3);
System.out.println("La suma de 5 + 3 es " + suma);
```

Pero existe una forma alternativa de implementar el interface funcional: con una **clase anónima**, esto es, una clase sin nombre que se define en el mismo momento de implementar el interface. Obsérvalo en este ejemplo:

```java
Operacion sumaAnonima = new Operacion() {
  @Override
  public double calcular(double a, double b) {
        return a + b;
  }
};
int suma = sumaAnonima.calcular(5,3);
System.out.println("La suma de 5 + 3 es " + suma);
```

Esto no representa, en principio, ninguna mejora. Solo hemos escrito el código de la clase Suma en otro sitio y hemos prescindido de su nombre. El siguiente paso para que esto resulte en una reducción notable de código es la expresión lambda.

## 5.3. Expresiones lambda

Una expresión lambda no es más que **una clase anónima escrita de forma superresumida** (chssst, no se lo digas a nadie).

Observa bien la clase anónima que hemos definido más arriba para implementar el interface funcional Operación. ¿Lo tienes? Pues ahora compáralo con el siguiente código, que es una implementación del mismo interface 100% equivalente, pero donde nos ahorramos volver a escribir el nombre del método (calcular()), el “new” de creación de la clase y, en realidad, casi todo el código:

```java
Operacion sumaLambda = (a, b) -> a + b;
int suma = sumaLambda.cacular(a, b);
System.out.println("La suma de 5 + 3 es " + suma);
```

¡En la primera línea ya lo hemos hecho todo\! Nos ahorramos gran parte del código de creación de la clase anónima y, en realidad, hacemos lo mismo: implementar el método `calcular()` (el único método abstracto) del interface `Operacion`.

Y esta es, en fin, la forma que tiene Java de “simular” la programación funcional: implementando interfaces funcionales, esto es, **interfaces con un solo método**. El resultado es bastante parecido al de un lenguaje funcional real.

### 5.3.1. Interfaces funcionales predefinidas en Java

Java ya incluye muchas interfaces funcionales listas para usar en el paquete **`java.util.function`**. Las más comunes son:

* **[`Predicate<T>`](https://docs.oracle.com/javase/8/docs/api/java/util/function/Predicate.html)**: su método **`test`** recibe un valor y devuelve un `boolean`. Puede ser muy útil para filtrar datos.

* **[`Consumer<T>`](https://docs.oracle.com/javase/8/docs/api/java/util/function/Consumer.html)**: su método **`accept`** recibe un valor y no devuelve nada. Es habitual a la hora de imprimir.

* **[`Function<T, R>`](https://docs.oracle.com/javase/8/docs/api/java/util/function/Function.html)**: usa el método **`apply`**, que recibe un valor de tipo `T` y devuelve un resultado de tipo `R`.

* **[`Supplier<T>`](https://docs.oracle.com/javase/8/docs/api/java/util/function/Supplier.html)**: su método **`get`** no recibe nada y devuelve un valor de tipo `T`.

¡Y hay **[muchas más](https://docs.oracle.com/javase/8/docs/api/java/util/function/package-summary.html)**!

Veámoslas en acción en el siguiente ejemplo, donde se define la clase `Alumno`, se crea una lista de objetos de ese tipo y luego se aplican una serie de reglas definidas mediante interfaces funcionales:

```java
import java.util.ArrayList;
import java.util.List;
import java.util.function.Consumer;
import java.util.function.Function;
import java.util.function.Predicate;

class Alumno {
    String nombre;
    double nota;
    
    Alumno(String nombre, double nota) {
        this.nombre = nombre;
        this.nota = nota;
    }
    
    @Override
    public String toString() {
        return nombre + " (Nota: " + nota + ")";
    }
}

public class Main {
    public static void main(String[] args) {
        // Generamos valores aleatorios
        Supplier<Double> nota = () ->
                (new Random()).nextDouble(0, 10.000001);
        
        List<Alumno> listaAlumnos = new ArrayList<>();
        listaAlumnos.add(new Alumno("Ana", nota.get()));
        listaAlumnos.add(new Alumno("Pedro", nota.get()));
        listaAlumnos.add(new Alumno("Luc�a", nota.get()));
        listaAlumnos.add(new Alumno("Juan", nota.get()));
        
        /*
        Definimos las Interfaces Funcionales con lambdas
        Predicate: ¿Tiene un sobresaliente (>= 9)?
        Predicate<Alumno> esSobresaliente = a -> a.nota >= 9.0;
        */
        
        // Function: dado un alumno, devuelve solo su nombre en mayúsculas
        Function<Alumno, String> obtenerNombreMayusculas =
            a -> a.nombre.toUpperCase();
        
        // Consumer: simplemente imprime un String
        Consumer<String> imprimir =
            s -> System.out.println("Candidato Premiado: " + s);
        
        System.out.println("--- Alumnos con Mención de Honor ---");
        procesarAlumnos(
            listaAlumnos,
            esSobresaliente,
            obtenerNombreMayusculas,
            imprimir
        );
    }
    
    /**
     * Método genérico que utiliza interfaces funcionales para
     * procesar datos.
     */
    public static void procesarAlumnos(
            List<Alumno> alumnos, 
            Predicate<Alumno> filtro, 
            Function<Alumno, String> transformador, 
            Consumer<String> accion) {
        
        for (Alumno a : alumnos) {
            if (filtro.test(a)) {
                String resultado = transformador.apply(a);
                accion.accept(resultado);
            }
        }
    }
}
```

En el código vemos cómo lanzamos los métodos de cada interfaz funcional (**`get`**, **`test`**, **`apply`** y **`accept`**), que ejecutarán las funcionalidades que hemos definido en las expresiones lambda.

Esto hace que nuestro código sea más modular, de forma que si mañana queremos filtrar por edad en lugar de por nota, **sólo cambiamos la expresión lambda**, dejando intacta la lógica del programa.

## 5.4. Programación funcional con Java y streams

Las expresiones lambda se usan mucho en Java combinadas con el **API streams**. Este API se usa para convertir una colección (por ejemplo, un List o un ArrayList) en un flujo de datos que pueden procesarse con muchos métodos funcionales predefinidos.

Mira estos ejemplos antes de que entremos en detalles sobre el API Streams:

```java
List<Integer> numeros = Arrays.asList(1, 2, 3, 4, 5);
int suma = numeros.stream()
                .filter(x -> x % 2 == 0)
                .sum();

// Imprime "Suma de números pares: 6"
System.out.println("Suma de números pares: " + suma);
```

En este ejemplo, se usa una expresión lambda para filtrar los números pares de una lista y luego se calcula la suma de los números pares.

Esos métodos que ves ahí, como `filter()` o `sum()`, son métodos del API Streams de Java, pensado específicamente para procesar colecciones de manera funcional.

Quizá se comprenda mejor cómo trabaja esto de la programación funcional con un ejemplo que compare este paradigma con la programación orientada a objetos convencional. Observa el siguiente código:

```java
public class FiltroNombres {
    public List<String> filtrar(List<String> nombres, String filtro) {
        List<String> nombresFiltrados = new ArrayList<>();
        for (String nombre : nombres) {
            if (nombre.startsWith(filtro)) {
                nombresFiltrados.add(nombre);
            }
        }
        return nombresFiltrados;
    }
}
```

Se trata de una clase, `FiltroNombres`, con un método `filtrar()` que recibe una lista de nombres y un String que funciona como filtro. Después, toma la lista de nombres y la recorre para buscar todos los nombres que comienzan por el String que actúa como filtro. Los nombres que cumplen la condición son añadidos a otra lista, que es la que el método devuelve.

Aquí tienes la versión funcional del mismo algoritmo:

```java
public static List<String> filtrarNombres(List<String> nombres, String filtro) {
  return nombres.stream()
      .filter(nombre -> nombre.startsWith(filtro))
      .collect(Collectors.toList());
}
```

Como puedes ver, el código es más compacto y se limita a invocar en el orden adecuado a otros métodos que ya existen y que, a su vez, usan programación funcional. 

## 5.5. El API Streams

El API Streams de Java permite trabajar de forma funcional con colecciones de datos, como, por ejemplo, arrays de todo tipo. 

De hecho, las operaciones con arrays son tan frecuentes, que se prestan mucho a usar programación funcional con ellas. ¿No es cierto que has recorrido muchas veces un array desde el principio hasta el final, aplicando alguna operación repetitiva a todos los elementos?

Esto de usar siempre un algoritmo similar (como el recorrido de un array) para resolver muchos problemas distintos se denomina **patrón de software**. Y los patrones de software pueden implementarse en métodos de la API para poder reutilizarlos sin tener que escribir sus algoritmos una y otra vez. Y así, de forma bastante natural, aterrizamos en la programación funcional.

Los métodos del API Streams se dividen en **intermedios** y **terminales**. Los intermedios se conocen como “perezosos” (*lazy*) porque no lanzan el proceso todavía, sino que permiten ir acumulando operaciones sobre la colección, **hasta que se invoca un método terminal**, que lanza el proceso, devuelve el resultado y cierra el Stream.

### Métodos intermedios

**Algunos métodos intermedios** habituales son:

**`filter(Predicate<T> predicate)`**  
Filtra (selecciona) los elementos del stream que cumplen una condición.
```java
List<String> filtered = list.stream()
                          .filter(s -> s.startsWith("A"))  
                          .collect(Collectors.toList());
```

**`map(Function<T, R> fun)`**  
Transforma cada elemento del stream.
```java
List<String> upperCaseList = list.stream()  
                               .map(String::toUpperCase)  
                               .collect(Collectors.toList());
```
  
**`distinct()`**  
Elimina los duplicados (usará el método `equals()` para determinar qué elementos del stream son iguales)
```java
List<String> distinctElements = list.stream()  
                                  .distinct()  
                                  .collect(Collectors.toList());
```

**`sorted()`**  
Ordena los datos. Puede indicarse el tipo de ordenación usando un método de la clase `Comparator`.
```java
List<String> sortedList = list.stream()
                              .sorted()
                              .collect(Collectors.toList());
```

**`takeWhile(Predicate<T> predicate)`**  
**`dropWhile(Predicate<T> predicate)`**  
Toma/desecha en cada caso los elementos que cumplan la condición del predicado. 

### Métodos terminales
De entre todos los **métodos terminales**, los más habituales son:

**`forEach(Consumer<T> consumer)`**  
Realiza la acción o acciones especificadas para cada elemento del stream.

```java
list.stream().forEach(System.out::println);
```

**`collect()`**  
Transforma los elementos en una colección u otro tipo de resultado.

```java
List<String> collectedList = list.stream()
                                 .collect(Collectors.toList());
```

**`toArray()`**  
Transforma los elementos en un array.

```java
String[] array = list.stream().toArray(String[]::new);
```

**`reduce()`**  
Reduce los elementos a un único valor.

```java
String reduced = list.stream().reduce((s1, s2) -> s1 + ", " + s2);
```

**`max()`** y **`min()`**  
Devuelve el máximo o el mínimo de entre los elementos del stream. Se puede indicar el mecanismo de comparación.

```java
int min = list.stream().min(Comparator.naturalOrder());
```

**`count()`**  
Cuenta el número de elementos del stream.

```java
long count = list.stream().count();
```
 
**`anyMatch(Predicate<T> p)`**, **`allMatch(Predicate<T> p)`**, **`noneMatch(Predicate<T> p)`**  
Verifica si algún, todos o ningún elemento del stream cumple con el predicado dado.

```java
boolean anyMatch = list.stream().anyMatch(s -> s.startsWith("A"));
```
 
**`findFirst()`**  
Devuelve el primer elemento del stream.

```java
String first = list.stream().findFirst();
```

## 5.6. Ventajas e inconvenientes de la programación funcional

Con estos sencillos ejemplos es posible que ya te hayas imaginado algunas de las **ventajas** de las expresiones lambda:

* **Código más compacto**: Permiten escribir, en teoría, código más conciso y fácil de leer.

* **Flexibilidad**: Se pueden usar en una variedad de contextos donde se hayan identificado patrones de software, como colecciones, streams y programación reactiva.

* **Menos efectos laterales**: el código funcional es más “cerrado” sobre sí mismo, lo que implica que es más difícil que un error en otra parte del programa pueda afectar a este código.

* **Eficacia**: Una vez que se controla este paradigma de programación, el desarrollo es más rápido y el tiempo de depuración se reduce.

Como todo en esta vida, las expresiones lambda también tienen sus **inconvenientes**:

* **Dificultad de depuración**: cuando algo falla, puede ser más difícil depurar el código funcional, precisamente porque es muy compacto.

* **Peor rendimiento**: parece ser que, en Java, el rendimiento del código funcional es algo peor que el del código tradicional. En otros lenguajes no tiene por qué ocurrir esto.

* **Curva de aprendizaje**: se requiere un poco de tiempo para familiarizarse con la sintaxis de la programación funcional.

Por último, tengo que decirte que las expresiones lambda y el API Streams no son la única característica de programación funcional que tiene Java. Desde la versión 8 en adelante, han ido apareciendo otros componentes de Java en la librería estándar que proporcionan muchas posibilidades para utilizar este paradigma de programación dentro de Java. 

Sin embargo, eso no convierte a Java en un lenguaje de programación funcional, sino que más bien se trata de parches que han conferido ciertas características funcionales a un lenguaje orientado a objetos. En cambio, hay otros lenguajes, como Python o Javascript, que, sin ser puramente funcionales, han integrado mejor el paradigma funcional en la propia naturaleza del lenguaje. 

Por último, debes saber que hay lenguajes de programación 100% funcionales, como Haskell o Erlang (muy usado en programación concurrente y en telecomunicaciones).
