## 6.1. El interface `Comparable`

En un tema anterior ya introdujimos la interfaz [**`Comparable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html)**. Ahora vamos a profundizar en ello y a compararlo con el interfaz [**`Comparator`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html)**, que hace algo parecido, es decir, permite comparar y ordenar objetos complejos, pero de manera diferente a Comparable.

Cuando empezamos a manejar colecciones (listas, conjuntos, etc.), surge una necesidad muy habitual: ordenar objetos. Es decir, establecer un orden entre ellos: cuál va antes, cuál va después. 

Con los números enteros, el orden está claro: el 1 va antes que el 2, el 2 antes que el 3, el 3 antes que el 995. Con las palabras, también hay un orden claro: el orden alfabético. Por eso Java ya sabe cómo ordenar y comparar objetos de tipo Integer o de tipo String.

Pero si creas tus propias clases, la cosa cambia. Imagina que tienes una clase `Alumno` que contiene nombre, apellidos, edad y curso. ¿Cómo debería ordenar Java los objetos de tipo Alumno? Es decir, dados dos alumnos cualesquiera, ¿cuál va antes y cuál después? ¿Los ordenamos alfabéticamente por nombre? ¿Por apellido? ¿Tal vez por edad o por curso?

La verdad es que todas esas ordenaciones tendrían sentido y serían válidas, y elegir una u otra dependerá de lo que necesitemos hacer. Pero una cosa está clara: Java no tendrá ni idea de cómo ordenarlas a menos que nosotros se lo digamos.

Y para hacer eso existen dos herramientas fundamentales en el JDK de Java:

* **El interfaz [`Comparable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html)**: se usa cuando la clase tiene un orden lógico principal.

* **El interfaz [`Comparator`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html)**: se usa cuando necesitas flexibilidad o hay múltiples formas de ordenar los objetos de una clase.

## 6.2. ¿Qué significa “comparar objetos”?

Comparar dos objetos consiste en establecer un orden entre ellos. Por ejemplo, ¿qué alumno tiene mayor nota?, ¿qué producto es más barato?, ¿qué libro va antes alfabéticamente?

En Java, esto se expresa con un método que devuelve:

* **Un número negativo** si el objeto actual es “menor”

* **El valor 0** si los dos objetos son iguales, es decir, si tienen el mismo orden

* **Un número positivo** si el objeto actual es “mayor”

Esto se ve muy claro con `String`. Observa este código:

```java
String s1 = "Charles";
String s2 = "Xavier";
String s3 = "Charles";
System.out.println(s1.compareTo(s2));  // Imprime -21 (negativo)
System.out.println(s2.compareTo(s1));  // Imprime 21 (positivo)
System.out.println(s1.compareTo(s3));  // Imprime 0
```

## 6.3. La interface `Comparable`

La interfaz [**`Comparable`](https://docs.oracle.com/javase/8/docs/api/java/lang/Comparable.html)** se usa cuando queremos definir un **orden natural** para una clase.

Esta interface se implementa en la propia clase y **solo permite un criterio de ordenación**. Obliga a implementar el método:

```java
int compareTo();
```

Por ejemplo, supongamos que tenemos una clase Alumno y que queremos ordenar los alumnos por nota. Podemos hacerlo así:

```java
public class Alumno implements Comparable<Alumno> {
    private String nombre;
    private double nota;
    
    // Aquí irían constructores, getters y setters
    @Override
    public int compareTo(Alumno otro) {
        if (this.nota < otro.getNota()) {
            return -1;
        } else if (this.nota > otro.getNota()) {
            return 1;
        } else {
            return 0;
        }
    }
}
```

Aquí estamos diciendo que un alumno es “mayor” que otro si tiene mayor nota, y estamos devolviendo un número negativo, un número positivo o un cero en cada caso, justo lo que se espera de `compareTo()`.

Ahora podemos usar ese comparador, por ejemplo, para ordenar una lista de alumnos usando el método **`ArrayList.sort()`**. ¡Si no hubiéramos implementado `compareTo()` en la clase `Alumno`, `sort()` no sabría cómo ordenarlos!

```java
List<Alumno> alumnos = new ArrayList<>();
alumnos.add(new Alumno("Ana", 8.5));
alumnos.add(new Alumno("Luis", 6.2));
alumnos.add(new Alumno("Marta", 9.1));

/*
Ordenamos los alumnos. null indica que debe ordenarse por el 
método “natural” definido en la clase Alumno
*/
alumnos.sort(null);

// Imprimimos la lista de alumnos ordenados por nota
for (Alumno a : alumnos) {
    System.out.println(a.getNombre() + " - " + a.getNota());
}
```

## 6.4. La interface `Comparator`

A veces nos interesa tener **varias formas de ordenar objetos de la misma clase**.

Por ejemplo, puede pasarnos con la clase `Alumno`. ¿Y si los queremos ordenar no solo por nota, sino también por nombre, o incluso por algo más rebuscado, como la longitud del nombre?

En estos casos, usamos el interface **[`Comparator`](https://docs.oracle.com/javase/8/docs/api/java/util/Comparator.html)** en lugar de `Comparable`.

El interfaz `Comparator` es externo a la clase y permite múltiples criterios de ordenación. En lugar del método `compareTo()`, define un método `compare()`, que recibe dos objetos del tipo que sea y los compara.

Por ejemplo, para comparar alumnos por su nombre tendremos que crear una clase especial que ejerza de “comparador de alumnos por nombre”:

```java
import java.util.Comparator;

public class ComparadorPorNombre implements Comparator<Alumno> {
    @Override
    public int compare(Alumno a1, Alumno a2) {
        return a1.getNombre().compareTo(a2.getNombre());
    }
}
```

Ahora podemos usar el nuevo comparador, por ejemplo, para ordenar una lista de alumnos por nombre:

```java
ArrayList<Alumno> alumnos = new ArrayList<>();
// Aquí rellenaríamos el array con alumnos
alumnos.sort(new ComparadorPorNombre());
```

Como ves, si método `ArrayList.sort()` recibe como parámetro un objeto de tipo “Comparador”, que implemente el interfaz `Comparator`, en lugar de un `null`, no usará el “método de ordenación natural” para ordenar, sino el objeto comparador pasado como parámetro.

Si construiremos otras clases comparadoras (`ComparadorPorEdad`, `ComparadorPorLongitudDelNombre` o lo que sea), podemos ordenar nuestra lista por cualquier criterio:

```java
alumnos.sort(new ComparadorPorNombre());
alumnos.sort(new ComparadorPorEdad());
alumnos.sort(new ComparadorPorLongitudDelNombre());
```

## 6.5. Comparadores con expresiones lambda

Desde Java 8, se puede simplificar mucho el código de los comparadores gracias a las **expresiones lambda**.

Un `Comparator` tradicional tiene este aspecto:

```java
alumnos.sort(new Comparator<Alumno>() {
    @Override
    public int compare(Alumno a1, Alumno a2) {
        return a1.getNombre().compareTo(a2.getNombre());
    }
});
```

(Hemos usado una clase anónima en lugar de una clase con nombre)

Pero Comparator\<Alumno\> no es más que un interface funcional con un solo método abstracto, compare(). Por lo tanto, puede resumirse en una expresión lambda:

```java
(a1, a2) -> a1.getNombre().compareTo(a2.getNombre())
```

Como el método `sort()` de un `ArrayList` espera un `Comparator<Alumno>`, podemos **alimentarlo con la expresión lambda** directamente:

```java
alumnos.sort((a1, a2) -> a1.getNombre().compareTo(a2.getNombre()));
```

## 6.6. Comparadores con `comparing()`

Hay una forma incluso más compacta que las expresiones lambda para comparar con `Comparator`: **el método `Comparator.comparing()`**.

Este comparador es equivalente a la expresión lambda anterior:

```java
alumnos.sort(Comparator.comparing(Alumno::getNombre));
```

`Comparator.comparing(...)` es un método de ayuda que recibe una función (lambda o referencia a método), extrae un valor de cada objeto y compara automáticamente esos valores. 

Por lo tanto, `Alumno::getNombre` es equivalente a la lambda `(a) -> a.getNombre()`, es decir, que este código:

```java
Comparator.comparing(Alumno::getNombre)
```

…significa literalmente: “Crea un comparador que compare alumnos usando su nombre”

## 6.7. Un ejemplo completo

Vamos a ver, para terminar esta sección, un ejemplo más realista donde usamos ambos enfoques (`Comparable` y `Comparator`). 

Usaremos una clase **`Producto`**. El “orden natural” de los productos será el precio, y lo implementaremos con el interfaz `Comparable`. Pero también haremos ordenaciones adicionales por otros criterios usando `Comparator` con expresiones lambda y con `comparing`:

```java
public class Producto implements Comparable<Producto> {
    private String nombre;
    private double precio;
    private int stock;
    
    public Producto(String nombre, double precio, int stock) {
        this.nombre = nombre;
        this.precio = precio;
        this.stock = stock;
    }
    
    public String getNombre() { return nombre; }
    public double getPrecio() { return precio; }
    public int getStock() { return stock; }
    
    // Orden natural: por precio
    @Override
    public int compareTo(Producto otro) {
        return Double.compare(this.precio, otro.precio);
    }
}
```

Observa ahora cómo comparamos productos en un programa que hace uso de la clase `Producto`, la ordena de forma “natural” (por precio) y de formas “no naturales” (por nombre y por stock descendente):

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        List<Producto> productos = new ArrayList<>();

        productos.add(new Producto("Teclado", 30.0, 10));
        productos.add(new Producto("Ratón", 15.0, 50));
        productos.add(new Producto("Monitor", 200.0, 5));

        // 1. Orden natural de la clase (por precio)
        Collections.sort(productos);

        System.out.println("Ordenados por precio:");
        for (Producto p : productos) {
            System.out.println(p.getNombre() + " - " + p.getPrecio());
        }

        // 2. Ordenamos por nombre usando lambdas
        productos.sort((p1, p2) -> p1.getNombre()
                                     .compareTo(p2.getNombre()));
	 // Expresión equivalente usando comparing():
        // productos.sort(Comparator.comparing(Producto::getNombre));

        System.out.println("\nOrdenados por nombre:");
        for (Producto p : productos) {
            System.out.println(p.getNombre());
        }

        // 3. Orden por stock descendente
        productos.sort((p1, p2) -> Integer.compare(
                        p2.getStock(),
                        p1.getStock()));
        // Expresión equivalente usando comparing():
        // productos.sort(Comparator.comparing(Producto::getStock.reversed());

        System.out.println("Ordenados por stock descendente:");
        for (Producto p : productos) {
            System.out.println(p.getNombre() + " - " + p.getStock());
        }
    }
}
```
