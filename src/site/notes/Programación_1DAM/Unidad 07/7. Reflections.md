Las **reflexiones** (**Reflections**) son un mecanismo de Java que permite inspeccionar y manipular clases, métodos, atributos y constructores en tiempo de ejecución, incluso aunque no se conozcan en tiempo de compilación. Vamos a introducirlas someramente en este apartado.

Hasta ahora hemos trabajado con clases, objetos y métodos que conocemos en **tiempo de compilación**. Por ejemplo:

```java
public class Persona {
    private String nombre;

    public Persona(String nombre) {
        this.nombre = nombre;
    }

    public void saludar() {
        System.out.println("Hola, soy " + nombre);
    }
}

Persona p = new Persona("Telmo");
p.saludar(); // Hola, soy Telmo
```

En este código, todo lo que hacemos se sabe en tiempo de compilación: sabemos que p es de tipo Persona y que tiene un método `saludar()`.

El mecanismo de “*reflection*” permite a Java inspeccionar clases, métodos, campos y constructores sin conocerlos en tiempo de compilación. Esto proporciona un enorme nivel de flexibilidad, más allá de la herencia y los interfaces, al permitir que los programas “se estudien a sí mismos” y actúen sobre esa información dinámicamente.

Entre otras cosas, eso hace posible:

* **Descubrir información de clases dinámicamente**. Por ejemplo, saber qué métodos o campos tiene una clase sin instanciarla de antemano.

* **Invocar métodos o acceder a campos dinámicamente**, incluso si son `private`. Esto es útil para frameworks que manejan objetos de manera genérica (como Spring o Hibernate).

* **Crear objetos de forma dinámica**. Es útil cuando no sabemos qué clase se usará hasta ejecutar el programa.

* **Metaprogramación**. Reflection nos permite escribir programas que trabajen con cualquier clase sin codificar nombres específicos.

Veamos algunos usos concretos de este mecanismo.

## 7.1. Obtener información de una clase

Las reflexiones siempre usan la **clase `Class`** como punto de partida.

Supongamos que tenemos una clase Persona. Usando una reflexión podemos inspeccionar su estructura sin instanciarla:

```java
Class<?> clase = Persona.class;   // ? es un símbolo comodín

System.out.println("Nombre de la clase: " + clase.getName());

System.out.println("Métodos:");
for (var metodo : clase.getDeclaredMethods()) {
    System.out.println(" - " + metodo.getName());
}

System.out.println("Campos:");
for (var campo : clase.getDeclaredFields()) {
    System.out.println(" - " + campo.getName());
}
```

Como ves, la clase Class tiene varios métodos útiles, como `getDeclaredMethods()` (devuelve un array con los nombres de los métodos) o `getDeclaredFields()` (lo mismo, con los campos). Así que el código anterior generará una salida de este estilo:

```
Nombre de la clase: Persona
Métodos:
 - getNombre
 - setNombre
 - saludar
 - <init>
Campos:
 - nombre
 - etc.
```

## 7.2. Crear objetos dinámicamente

Supongamos que no sabemos qué clase vamos a instanciar hasta que se ejecute el programa. ¿Cómo podrías instanciar objetos de una clase que no conoces en tiempo de compilación?

A partir del nombre de la clase (que puede estar almacenado en un String) podemos crear un objeto de ese tipo y lanzar su constructor o cualquier otro método:

```java
System.out.println("Dime qué clase quieres instanciar");
String nombreClase = System.console().readLine();

Class<?> clase = Class.forName(nombreClase);
Constructor<?> constructor = clase.getConstructor(clase.class);
Object objeto = constructor.newInstance("Luis");

Method metodoSaludar = clase.getMethod("saludar");
metodoSaludar.invoke(objeto);
```

Si a este programa le decimos que queremos instanciar la clase `Persona`, podríamos obtener una salida como esta:

`Hola, soy Luis`

Observa que aquí no usamos `new Persona("Luis")` directamente, sino que creamos la instancia con `newInstance()` e invocamos los métodos con `invoke()`.

## 7.3. Acceder a campos privados

Reflection permite incluso romper el encapsulamiento, lo cual es útil en algunos contextos, pero también peligroso. (¡No hagan esto en sus casas, niños y niñas! ¡Puede explotar!)

```java
Persona p = new Persona("Ana");
Field campoNombre = Persona.class.getDeclaredField("nombre");
campoNombre.setAccessible(true); // ¡¡Permite acceso a private!!
campoNombre.set(p, "Telmo");

p.saludar(); // Hola, soy Telmo
```

Esto nos muestra el poder (y el riesgo) de Reflection: podemos cambiar incluso lo que debería ser inaccesible.

## 7.4. Métodos útiles de la clase `Class`

Como has visto en estos ejemplos, la clase **Class** tiene un montón de **métodos útiles** para usar las reflexiones. Aquí te dejo una lista de los más habituales:

Métodos para obtener información general de una clase

* getName() → devuelve el nombre completo (incluye paquete)

* getSimpleName() → devuelve solo el nombre de la clase

* getPackageName() → devuelve el nombre del paquete

* getSuperclass() → devuelve la superclase

* getInterfaces() → devuelve las interfaces implementadas

* getModifiers() → devuelve los modificadores (public, abstract, etc.)

Métodos para acceder a los métodos de una clase:

* getMethods() → métodos públicos (incluye heredados)

* getDeclaredMethods() → todos los métodos (incluye privados, no heredados)

* getMethod(String nombre, Class\<?\>... params) → método público concreto

* getDeclaredMethod(String nombre, Class\<?\>... params) → método concreto (incluye privados)

Métodos para acceder a los campos (atributos) de una clase:

* getFields() → campos públicos (incluye heredados)

* getDeclaredFields() → todos los campos (incluye privados)

* getField(String nombre) → campo público concreto

* getDeclaredField(String nombre) → campo concreto (incluye privados)

Métodos para acceder a los constructores de una clase:

* getConstructors() → constructores públicos

* getDeclaredConstructors() → todos los constructores

* getConstructor(Class\<?\>... params) → constructor público concreto

* getDeclaredConstructor(Class\<?\>... params) → constructor concreto (incluye privados)

Métodos para crear objetos mediante Reflection:

* getConstructor(...).newInstance(...) → crear instancia con constructor específico

* getDeclaredConstructor(...).newInstance(...) → incluso si es privado

Métodos para invocar y manipular objetos creados con Reflection:

* Obtener Method → luego usar invoke(obj, args...)

* Obtener Field → luego usar get(obj) / set(obj, valor)

Método para cargar dinámicamente una clases:

* Class.forName(String nombreClase) → carga una clase por nombre

Métodos de comprobación de tipos:

* isInterface() → si es una interfaz

* isEnum() → si es un enum

* isArray() → si es un array

* isPrimitive() → si es tipo primitivo

* isAnnotation() → si es una anotación

Métodos para comprobar relaciones entre clases:

* isAssignableFrom(Class\<?\> otraClase) → comprobar herencia/polimorfismo

## 7.5. Precauciones al usar Reflection

Como todo en esta vida, las reflexiones de Java son una característica muy poderosa que nos proporciona un control muy grande sobre la ejecución de nuestros programas pero, sin embargo, no está exenta de ciertos problemas. Estos son los principales:

* **Rendimiento**: Reflection es más lento que el acceso normal a métodos/propiedades.

* **Seguridad**: como se viola el encapsulamiento (`private`), podemos tocar datos sensibles y romper objetos que de otro modo funcionarían bien.

* **Mantenimiento**: los errores aparecen en tiempo de ejecución, no en compilación, y eso siempre es más costoso de depurar.
