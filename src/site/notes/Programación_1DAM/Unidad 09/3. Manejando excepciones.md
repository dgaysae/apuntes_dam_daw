## 3.1. `SQLException`

En el paquete **`java.sql`** se encuentra la clase **`SQLException`** que captura las excepciones ocurridas en el manejo de la base de datos. Su uso no difiere del resto de excepciones, pero incorpora algunos métodos específicos interesantes: 

* **`getSQLState()`**. Describe el error ocurrido según las convenciones `XOPEN`. 

* **`getMessage()`**. El método típico de excepciones, salvo que éste recoge el texto que envía el controlador JDBC según lo informado por el gestor de bases de datos, lo que le hace muy efectivo.

* **`getErrorCode()`**. Devuelve el código de error ocurrido (según lo informado por el gestor de la base de datos) 

* **`getNextException()`**. Permite ver la siguiente excepción ocurrida, ya que a veces ocurren varias a la vez (especialmente en transacciones y operaciones complejas). El siguiente ejemplo ilustra este caso:

```java
try ( ... ) {

}   
catch (SQLException e) {  
    while (e != null) {  
        System.err.println("Estado: " + e.getSQLState());  
        System.err.println("Código: " + e.getErrorCode());  
        System.err.println("Mensaje: " + e.getMessage());  
        e = e.getNextException(); // Esencial para no crear un bucle infinito
    }  
}
```

## 3.2. `SQLWarning`

A veces, ocurre que la base de datos no devuelve un error sino una advertencia (warning). Las advertencias son errores de menor gravedad, esto es, no impiden la ejecución de una sentencia, pero como errores que son deberían ser revisados y corregidos.

Los objetos que provocan los warnings (sobre todo los `ResultSet`) las van almacenando en objetos de tipo **`SQLWarning`**. Para revisar estos errores basta con llamar repetidamente al método **`getSQLWarning`**. En cada llamada obtendremos un nuevo objeto `SQLWarning` hasta que, finalmente, no haya más (devolverá entonces el valor `null`). 

Los métodos que permiten observar el contenido de la advertencia son los mismos que los de la clase **`SQLException`**, ya que **`SQLWarning`** deriva de ella. Añade el método **`getNextWarning()`** que funciona de forma similar a `getNextException()`, pero que en esta clase no se suele utilizar ya que las sucesivas llamadas al método **`getSQLWarnings()`** proporcionan el mismo resultado. 

El método **`clearWarnings()`** de la clase `ResultSet` permite borrar las advertencias almacenadas hasta ese momento.
