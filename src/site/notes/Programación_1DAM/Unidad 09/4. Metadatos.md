## 4.1. ¿Metaqué?

Hay casos en los que se requiere conocer la estructura de una base de datos (nombre y diseño de las tablas, tipos de los campos, etc.). Los datos que describen la estructura de las bases de datos es lo que se conoce como **metadatos**. 

Los metadatos se obtienen utilizando el método `getMetaData()` de la clase `Connection`, por lo que es el objeto de la conexión el que permite obtener estos metadatos. El resultado de este método es un objeto de clase **`DatabaseMetaData`**. 

```java
// con es un objeto Connection 
DatabaseMetaData metadatos = con.getMetaData();
```

La clase DatabaseMetaData contiene multitud de métodos para acceder a los metadatos de nuestra BD. Aquí vemos solo algunos de ellos:

* String **`getDatabaseProductName()`**: Devuelve el nombre comercial del SGBD utilizado.

* String **`getDatabaseProductVersion()`**: Devuelve el número de versión del SGBD actual.

* String **`getDriverName()`**: Devuelve el nombre del driver JDBC.

* int **`getMaxColumnNameLength()`**: Devuelve el máximo número de caracteres que puede poseer un campo en esta base de datos.

* int **`getMaxColumnsInIndex()`**: Devuelve el número máximo de campos que pueden colocarse en un índice en este SGBD.

* String **`getSQLKeywords()`**: Devuelve una cadena con la lista de todos los comandos SQL, separados por comas, reconocidos por este SGBD.

* String **`getURL()`**: devuelve la URL del SGBD.

* boolean **`isReadOnly()`**: indica si la BD seleccionada es de solo lectura.

* boolean **`supportsANSI92FullSQL()`**: indica si el SGBD soporta completamente el estándar SQL 92.

* ResultSet **`getTables()`**: obtiene un `ResultSet` en el que cada tupla es una tabla del SGBD. Otros métodos como `getColumns()` o `getPrimaryKeys()` nos permiten conocer los campos de esas tablas y las claves primarias.

## 4.2. Metadatos de una consulta

Si aplicamos el método `getMetaData()` a la clase `ResultSet`, obtendremos como resultado un objeto de tipo 

`ResultSetMetaData`, que nos proporcionará información sobre el conjunto de resultados obtenido. Información tan útil como esta: 

* **`int getColumnCount()`**: el número de columnas de la consulta.

* **`String getColumnName(int n)`**: el nombre (en la BD) de la columna número `n`

* **`int getColumnType(int n)`**: el tipo de datos SQL de la columna `n`. 

* **`int getColumnTypeName(int nColumna)`**: el tipo de datos compatible con Java de la columna `n`.

* **`boolean isAutoincrement(int n)`**: devuelve true si la columna `n` es autoincrementable.

* **`boolean isCaseSensitive(int n)`**: devuelve true si la columna `n` es sensitiva a las mayúsculas. 

* **`boolean isReadOnly(int n)`**: devuelve true si la columna `n` es de solo lectura.
